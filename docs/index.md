# VistA Evolved Platform — Documentation

Welcome to the VistA Evolved Platform documentation.

## Documentation model

This site follows a strict documentation model with six approved categories:

| Category | Purpose |
|----------|---------|
| [Reference](reference/index.md) | Policies, registries, indexes, technical references |
| [Explanation](explanation/index.md) | Architecture rationale, governance protocols |
| [ADRs](adrs/index.md) | Architecture decision records |
| [Runbooks](runbooks/index.md) | Operational procedures and reports |
| [How-To](how-to/index.md) | Task-oriented guides |
| [Tutorials](tutorials/index.md) | Step-by-step learning paths |

## Key governance files

- [Source of Truth Index](reference/source-of-truth-index.md) — where canonical definitions live
- [Doc Governance](reference/doc-governance.md) — documentation rules and approved categories
- [Decision Index](reference/decision-index.yaml) — enterprise-namespaced ADR registry
- [AI Coding Governance](explanation/ai-coding-governance-and-sdlc.md) — operating rules for AI agents

## Principles

- **Terminal-first.** VistA truth before GUI.
- **Contract-first.** Specs before implementation.
- **Proof required.** No claiming done without evidence.
- **One slice at a time.** Verify before expanding.
