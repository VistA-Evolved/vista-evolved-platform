# Boundary policy

- **Bounded contexts** are reflected in directory layout and CODEOWNERS: `apps/control-plane/`, `apps/control-plane-api/`, `apps/tenant-admin/`, `packages/contracts/`, `packages/config/`, `packages/domain/`, `packages/ui/`.
- **Cross-boundary access** is via contracts (OpenAPI, AsyncAPI, schemas). No direct imports from one app into another’s internals; use shared packages and generated clients.
- **Domain** packages (`packages/domain/admin/`, `packages/domain/tenancy/`) hold shared types and rules; they do not depend on apps or infrastructure. Apps depend on domain and contracts.- **VistA adapter boundary**: `apps/tenant-admin/lib/vista-adapter.mjs` is the sole entry point for VistA data in the tenant-admin workspace. It delegates to `xwb-client.mjs` (XWB RPC broker protocol over TCP). No other app or package may import the XWB client directly. The adapter returns normalized objects with `source: 'vista'`; callers never see raw RPC frames.
