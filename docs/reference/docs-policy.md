# Docs policy

Only the following doc categories are canonical:

- **tutorials/** — Step-by-step learning paths.
- **how-to/** — Task-oriented guides.
- **reference/** — Port registry, contract system, source-of-truth index, data ownership, boundary/contract/persistence policies.
- **explanation/** — Governed build protocol, architecture rationale.
- **adrs/** — Architecture decision records.
- **runbooks/** — Operational procedures.

No broad documentation sprawl. No duplicate of prompt content in docs. Verification outputs and evidence go under `artifacts/`, not docs.
