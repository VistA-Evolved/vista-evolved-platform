# 1. Purpose
This is the third truth-layer artifact. It delivers what the design team and AI coding team have been waiting for: exact page inventories and screen contracts for Wave 1 workspaces.
For each page, this document specifies: purpose, primary user role, every field on the page, every button and action, every dropdown with its data source, every tab, every state (empty, loading, error, success, conflict, permission-denied), every warning/confirmation dialog, and the exact backend touchpoint (existing RPC or required custom wrapper with the VistA file/field reference).
This volume covers Admin/Security (User and Provider Management) and Scheduling. These two workspaces are the most critical because: (1) user/provider setup is a prerequisite for every other workspace, and (2) scheduling has the strongest RPC coverage and is the safest first build target.

### Create Staff: Backend Action Chain
The ZVE USER CREATE wrapper must perform the following in sequence:
## 1. Create new entry in NEW PERSON #200 with NAME, SEX, DOB, SSN/identifier.
## 2. Set ACCESS CODE and VERIFY CODE (may be system-generated initial values).
## 3. Set DIVISION multiple entries.
## 4. Set PRIMARY MENU OPTION #201 (derived from role selection).
## 5. Set SECONDARY MENU OPTIONS #203.
## 6. Set SERVICE/SECTION #29.
## 7. Assign security keys via KEYS multiple.
## 8. If provider: set Person Class, NPI, DEA, Authorized to Write Meds, pharmacy-specific fields.
## 9. Set SIGNATURE BLOCK PRINTED NAME #20.2 (auto-derived from name).
## 10. Return: new user IEN, success/failure status, any validation errors.
CRITICAL: The wrapper must validate all inputs against VistA field transforms. For example, NAME must pass the #200 input transform (3-35 chars, uppercase, LAST,FIRST format). DEA must pass pharmacy validation. Access/verify codes must meet Kernel length/complexity requirements.
