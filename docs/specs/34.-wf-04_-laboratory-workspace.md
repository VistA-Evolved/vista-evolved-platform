WF-04: Laboratory Workspace
Document 4 of 16 | ~30 Pages | Clinical Lab + Blood Bank (VBECS) + Anatomic Pathology
March 2026 | The Laboratory Workspace unifies three lab sub-domains: (A) Clinical Laboratory (Vol 6) — chemistry, hematology, coagulation, urinalysis, microbiology, point-of-care testing. Order accessioning, specimen collection/tracking, result entry/verification, instrument interfacing via AUTO INSTRUMENT #62.4, QC management. (B) Blood Bank / VBECS (Vol 6 Supplement Section A) — patient ABO/Rh typing, antibody screening, crossmatching, blood product inventory, transfusion management. VBECS is a SEPARATE Windows/Azure application — Lab workspace provides the VistA-side integration views. (C) Anatomic Pathology (Vol 6 Supplement Section B) — surgical pathology, cytopathology, electron microscopy, autopsy. Specimen accession, grossing, microscopic examination, SNOMED coding, CAP synoptic cancer reporting, report sign-out.

LB-01
Lab Pending Orders
Worklist (WF-01 5.1)
Lab orders awaiting processing. Columns: Patient, Test/Panel Ordered, Specimen Type, Urgency (STAT red badge, ASAP amber, Routine gray), Ordering Provider, Order Date/Time, Accession Status (Not Accessioned / Accessioned / Collected / Received in Lab). Filter: urgency, test type (Chemistry/Hematology/Coag/UA/Micro), date range, ordering location. Actions: Accession, Print Labels, Cancel.
ORDER #100 lab orders + LABORATORY TEST #60 via ZVE LAB ORDERS wrapper.
STAT lab orders pinned to top with red left border + pulsing badge. Accession action assigns accession number + prints specimen labels with barcode. Label includes: patient name, MRN, accession#, test(s), collection time, specimen type, barcode for instrument loading. Batch accession: select multiple orders, accession all at once.
LB-02
Specimen Collection / Tracking
Worklist (WF-01 5.1) + Form
Collection worklist: patients needing specimen collection, sorted by location (ward -> room -> bed) for phlebotomy rounds. Columns: Patient, Location, Test(s), Specimen Type (tube color indicator: purple=EDTA, blue=citrate, green=lithium heparin, red/gold=SST, gray=sodium fluoride), Collection Status (Pending/Collected/In Transit/Received). Collection form: scan patient wristband, scan tube barcode, record collection time, collector ID.
Lab specimen tracking data via ZVE LAB SPECIMEN wrapper.
Tube color indicators: visual reference for phlebotomist showing correct tube type per test. Patient wristband scan: verifies correct patient before collection (patient safety — wrong-patient specimen is a critical lab error). Specimen rejection tracking: hemolyzed, clotted, insufficient quantity, wrong tube, mislabeled — with rejection reason and recollection request notification.
LB-03
Result Entry / Verification
Form (WF-01 5.3) + Worklist
Split view: Left: worklist of tests awaiting results (from instrument auto-filing or manual entry). Right: result entry form for selected test. Fields: Result Value (numeric or coded), Units, Reference Range (auto-displayed), Interpretive Comment, Flag (auto-calculated: Normal/High/Low/Critical based on reference range). Technologist enters/verifies result. Pathologist/supervisor signs critical values.
LABORATORY TEST #60 result sub-files via ZVE LAB RESULTS wrapper. AUTO INSTRUMENT #62.4 for instrument-filed results.
AUTO-VERIFICATION: instrument results that fall within normal reference ranges and pass delta check (comparison to prior result) can be auto-verified without manual review per site-configured auto-verify rules. MANUAL VERIFICATION: required for abnormal, critical, and flagged results. CRITICAL VALUE NOTIFICATION: when a critical result is verified, system auto-generates notification to ordering provider (per CLIA regulations — critical value must be communicated to provider within defined timeframe).
LB-04
Verified Results / Reporting
Detail View (WF-01 5.2)
Patient lab results display (same view as CL-07 in Clinical workspace but with additional lab-side detail): result + reference range + flag + verifying tech + verification time + instrument used + specimen quality notes. Report generation: formatted lab report for printing/faxing to external providers. Result amendment: correction workflow with original value preserved + amendment reason.
LABORATORY TEST #60 verified results via ZVE LAB REPORT wrapper.
Result amendment: if verified result is found to be incorrect, amendment creates addendum (original value + amended value + reason) per Audit/Irreversibility Matrix (C/L3). Amended results flagged in patient chart with 'Corrected' indicator. LOINC code displayed alongside local test name for interoperability.
LB-05
Instrument Interface Status
Dashboard (WF-01 5.4)
Dashboard of all connected lab instruments: Instrument Name, Type (Chemistry/Hematology/Coag/etc.), Connection Status (Online green/Offline red/Degraded amber), Last Communication timestamp, Message Throughput (msg/hr), Error Rate (%), Queue Depth (pending results). Actions per instrument: View Log, Restart Connection, Test Connection. Alarm panel: instruments with errors or offline status.
AUTO INSTRUMENT #62.4 status via ZVE LAB INSTRUMENT wrapper.
Real-time monitoring: connection status polls every 30 seconds. Offline instrument: RED alert + notification to lab supervisor + instructions for manual result entry workaround. Error rate threshold: >5% triggers investigation alert. Message log: shows individual HL7/ASTM messages sent/received for troubleshooting. Device Catalog (Phase 2, Artifact 8) reference for supported instruments.
LB-06
Auto-Verify Rules Management
Config (WF-01 5.6)
Configuration table: Test Name, Auto-Verify Enabled (Y/N), Normal Range for Auto-Verify (may differ from reference range), Delta Check Limits (maximum acceptable change from prior result), Additional Rules (e.g. 'No auto-verify if patient age < 18', 'No auto-verify if specimen hemolyzed'). Enable/Disable per test. Audit log of rule changes.
Lab auto-verify configuration via ZVE LAB AUTOVERIFY wrapper.
CAUTION banner. Auto-verify rules directly impact patient safety: over-permissive rules may release erroneous results without human review. Under-permissive rules create unnecessary manual verification workload. Rule changes require Lab Director or designee approval. Every rule change audit-logged.
LB-07
Quality Control Dashboard
Dashboard (WF-01 5.4)
QC monitoring: per instrument, per analyte. Levey-Jennings charts: QC values plotted over time with mean and +/-1SD, +/-2SD, +/-3SD lines. Westgard rule violation flags: 1-2s warning, 1-3s rejection, 2-2s, R-4s, 4-1s, 10x rules. QC status: In Control (green), Warning (amber), Out of Control (red — all patient results since last in-control QC must be reviewed).
Lab QC data via ZVE LAB QC wrapper.
Levey-Jennings chart is the standard QC visualization for clinical laboratories. Westgard rules are the standard decision criteria. Out-of-control status: system blocks result release for affected instrument/analyte until QC is re-established and results are reviewed. CLIA requirement: QC performed and documented per test system manufacturer requirements.
LB-08
Lab Workload / Productivity
Report (WF-01 5.5)
Reports: test volumes by section (Chemistry/Hematology/Coag/UA/Micro/POCT), turnaround times (order to result, collection to result, receipt to result — target times configurable), STAT turnaround compliance, critical value notification compliance, QC documentation compliance, specimen rejection rates, auto-verify rates. Workload coding: CAP workload units per test.
Lab workload data via ZVE LAB WORKLOAD wrapper.
Standard Report template. Key regulatory metrics: STAT turnaround (order to result < 60 min for Chemistry/Hematology), critical value notification (< 30 min from verification to provider communication), specimen rejection rate (target < 2%).
LB-09
Microbiology Worklist
Worklist (WF-01 5.1) SPECIALIZED
Micro-specific worklist: cultures in progress. Columns: Patient, Specimen Type (blood, urine, wound, sputum, CSF), Collection Date, Days Incubating, Preliminary Result (No Growth/Growth — organism ID in progress), Final Result (organism + sensitivities), Status (Incubating/Preliminary/Final/No Growth). Antibiogram display: organism x antibiotic sensitivity matrix (S/I/R color-coded).
Lab micro results via ZVE LAB MICRO wrapper.
Microbiology results are multi-level (unlike discrete chemistry values): Culture -> Organism ID -> Antibiotic Sensitivities (MIC values + interpretation S/I/R). Preliminary reports: Gram stain results reported within hours. Final reports: organism ID + sensitivities may take 24-72 hours. Antibiogram: facility-level cumulative antibiotic susceptibility data for empiric therapy guidance.
LB-10
Point-of-Care Testing (POCT)
Dashboard (WF-01 5.4) + Worklist
POCT device management: registered devices (glucometers, i-STAT, etc.), operator competency status, QC compliance per device, pending results for review. Result worklist: Patient, Test, Value, Device, Operator, Time, Review Status (Pending Review / Reviewed / Flagged). POCT coordinator reviews results for accuracy.
Lab POCT data via ZVE LAB POCT wrapper.
POCT1-A device integration per Device Catalog. Operator competency tracking: each POCT operator must demonstrate annual competency. QC required before patient testing on each device per shift. Patient ID verification at device: glucometer scans patient wristband before accepting specimen (prevents wrong-patient POCT results).
LB-11
Lab Reports Hub
Report (WF-01 5.5)
Aggregated laboratory reports: volumes, turnaround, QC, workload, specimen rejection, auto-verify rates, POCT compliance, critical value compliance, personnel workload distribution, instrument utilization. Exportable for CAP/CLIA accreditation documentation.
Various ZVE LAB wrappers.
Standard Report template. CAP/CLIA accreditation support: reports formatted to meet accreditation documentation requirements. QC records exportable for inspector review.
LB-12
Lab Configuration
Config (WF-01 5.6)
Configuration: LABORATORY TEST #60 setup (test definitions, reference ranges by age/sex, critical values), AUTO INSTRUMENT #62.4 connections (instrument name, protocol, connection parameters), specimen type definitions, collection requirements, result format templates, accession area setup, POCT device registration, QC parameters.
ZVE LAB CONFIG wrapper -> Lab configuration files.
CAUTION banner. Test definition changes affect all result interpretation. Reference range changes: must be clinically validated. Critical value thresholds: changes require Lab Director approval and clinical staff notification. Instrument connection changes: require coordination with biomedical engineering.

BB-01
Blood Product Order
Form (WF-01 5.3)
CPRS-integrated order form. Fields: Product Type (dropdown: Packed RBCs, FFP, Platelets, Cryoprecipitate, Whole Blood — with icons), Quantity (units), Urgency (STAT/Urgent/Routine), Clinical Indication (ICD-10 search: anemia, hemorrhage, coagulopathy, thrombocytopenia, surgery), Diagnostic Tests (checkboxes: Type & Screen, Crossmatch). Type & Screen currency check: if no T&S within 3 days, system blocks component ordering and prompts T&S order.
ZVE BB ORDER wrapper -> ORDER #100 + VBECS via VistALink. CPRS-VBECS Interface (OR*3.0*212).
Type & Screen 3-day currency rule enforced automatically. Coordinated order dialog: blood component + diagnostic test ordered together in single workflow. MSBOS (Maximum Surgical Blood Order Schedule) reference: for surgical patients, suggested crossmatch quantity displayed based on procedure type (Vol 12 Surgery integration).
BB-02
Type & Screen Results
Detail View (WF-01 5.2)
Patient blood type display: ABO Group (large text: A, B, AB, O) + Rh Type (Pos/Neg) with color-coded badge (Rh-Neg = yellow caution badge). Historical type comparison: current vs prior (discrepancy = RED alert). Antibody Screen: Negative (green) or Positive (red + identified antibody list: Anti-Kell, Anti-Duffy, etc.). Special Instructions / Transfusion Requirements from VBECS.
ZVE BB RESULTS wrapper -> VBECS patient testing data.
ABO/Rh DISCREPANCY between current and historical: CRITICAL patient safety alert — may indicate wrong patient, sample swap, or new alloantibody development. Red alert banner blocks further blood bank actions until resolved by Blood Bank Medical Director. Antibody identification results: drive crossmatch compatibility requirements (antigen-negative units required).
BB-03
Crossmatch Status / Unit Assignment
Worklist (WF-01 5.1)
Table of blood units assigned to patient: Unit ID (ISBT 128 barcode), Product Code, ABO/Rh, Expiration Date/Time, Special Attributes (CMV-Neg, Irradiated, Leukoreduced, Washed — as icon badges), Crossmatch Type (Electronic/Serologic), Status (Assigned/Issued/Returned/Transfused/Expired). Actions: Issue (to requesting location), Return (unused unit back to inventory), Cancel Assignment.
ZVE BB UNITS wrapper -> VBECS unit data.
ISBT 128 coding displayed alongside human-readable text. Unit expiration countdown: amber when <24 hours, red when <4 hours. Special attributes as colored icon badges for quick visual identification. Issue action requires: destination location, receiving person name, verification of unit label vs patient ID.
BB-04
Transfusion Record
Form (WF-01 5.3) SPECIALIZED
Bedside transfusion documentation. Sections: (1) Pre-Transfusion: patient ID verification (wristband scan), unit label verification (unit barcode scan), vital signs (Temp, BP, HR, RR — from Vitals #120.5 or manual entry), blood warmer used (Y/N). (2) During Transfusion: vital signs Q15min for first hour then Q30min (mini-flowsheet grid), infusion rate, volume infused, any symptoms (fever, chills, rash, dyspnea, back pain). (3) Post-Transfusion: completion time, total volume, post-transfusion vitals, reaction (Y/N — if Y triggers BB-05). Nurse documentation.
ZVE BB TRANSFUSE wrapper -> VistA clinical record.
TWO-PERSON bedside verification: two nurses independently verify (1) patient wristband = order patient, (2) unit label ABO/Rh = patient ABO/Rh, (3) unit ID matches crossmatch record. Both verifiers sign electronically. Vital sign monitoring grid: time-column entry similar to mini ICU flowsheet. Reaction monitoring: any symptom triggers 'STOP TRANSFUSION' protocol prompt.
BB-05
Transfusion Reaction Investigation
Form (WF-01 5.3)
Reaction documentation: Reaction Type (dropdown: Febrile Non-Hemolytic, Allergic/Urticarial, Anaphylactic, Acute Hemolytic, Delayed Hemolytic, TRALI, TACO, Bacterial Contamination, Other), Severity (Mild/Moderate/Severe/Fatal), Time of Onset, Symptoms, Vital Signs at Reaction, Workup Results (DAT, visual hemolysis check of pre/post specimens, clerical check, culture of unit), Outcome, Reporting (FDA MedWatch if applicable).
ZVE BB REACTION wrapper -> VBECS reaction workup + VistA clinical data.
Reaction triggers: (1) STOP transfusion immediately, (2) notify Blood Bank, (3) return implicated unit + post-transfusion specimen to BB, (4) Blood Bank performs workup in VBECS. Results flow back to this page. Severe reactions: auto-generate incident report. Fatal reactions: FDA MedWatch reporting mandatory.
BB-06
Blood Inventory Dashboard
Dashboard (WF-01 5.4)
KPI cards: Total Units On Hand, Units Expiring <24h (red count), Units Crossmatched/Reserved (amber), Units Available (green). Main grid: inventory table by Product Type x ABO/Rh. Rows: Packed RBCs, FFP, Platelets, Cryo. Columns: O-Neg (universal), O-Pos, A-Neg, A-Pos, B-Neg, B-Pos, AB-Neg, AB-Pos. Cell values: count. Color: green (adequate), amber (low), red (critical shortage).
ZVE BB INVENTORY wrapper -> VBECS inventory data.
O-Negative is UNIVERSAL DONOR for RBCs — always highlighted as critical supply. Expiring units alert: amber <72h, red <24h — drives usage prioritization (oldest first). Critical shortage notification: if any product/type falls below minimum par level, alert to Blood Bank supervisor + potential activation of emergency blood supply protocols. Real-time updates from VBECS.
BB-07
Blood Bank Reports
Report (WF-01 5.5)
Reports: transfusion statistics (units transfused by product/type/service), crossmatch-to-transfusion ratio (C:T — target <2.5:1, indicates over-ordering if higher), wastage (expired/damaged/returned units by reason), reaction rates (per 100 transfusions by type), inventory turnover, MSBOS compliance (actual crossmatch vs schedule recommendation), Type & Screen turnaround times.
ZVE BB REPORTS wrapper -> VBECS reporting data.
Standard Report template. Key quality metric: C:T ratio — high ratio means blood is being crossmatched but not transfused (waste of blood bank resources and potentially denying units to other patients). Reaction rates: benchmarked against national AABB standards.
BB-08
Blood Bank Configuration
Config (WF-01 5.6)
Configuration: VBECS connection parameters (VistALink settings), division settings, electronic crossmatch enable/disable, T&S currency period (default 3 days, configurable), MSBOS table (procedure -> crossmatch quantity), transfusion reaction reporting parameters, inventory par levels by product/type.
ZVE BB CONFIG wrapper -> VBECS + VistA BB configuration.
CAUTION banner. VBECS configuration: most BB configuration done within VBECS GUI itself — this page manages the VistA-side integration parameters. Electronic crossmatch enable: per FDA and AABB requirements, requires validation before activation. MSBOS table: developed by transfusion committee, reviewed annually.

CAP synoptic gross elements: for cancer specimens, structured data capture for specimen integrity, margin inking colors, lymph node counts. Block tracking: each cassette/block logged for downstream processing in histology lab.
AP-03
Microscopic Examination & Diagnosis
Form (WF-01 5.3) + Detail View SPECIALIZED
Pathologist sign-out workstation. Left 50%: Rich text editor for microscopic description + diagnosis entry. Diagnosis fields: Primary Diagnosis (free text), SNOMED Code(s) (topography + morphology searchable picker), ICD-O-3 Code (for tumors — morphology + behavior), ICD-10-CM (diagnostic code). Comment field. Right 50%: Digital slide viewer (if available — integrated whole slide image display via DICOM SM or vendor viewer) OR 'View on Microscope' placeholder. Below: Case summary showing: gross description, clinical history, prior pathology for this patient, immunohistochemistry results.
ZVE AP MICRO wrapper -> LR AP microscopic/diagnosis fields. Digital pathology via VistA Imaging DICOMweb.
SNOMED code picker: type topography (e.g. 'Breast') -> shows relevant morphology codes (e.g. 'Infiltrating Duct Carcinoma'). Auto-suggests based on specimen type. ICD-O-3 for cancer cases: morphology code + behavior code (/0 benign, /1 borderline, /2 in situ, /3 malignant). Case comparison: prior AP cases for same patient displayed for pathologist reference. Immunohistochemistry results: from special stain orders, displayed alongside morphology for integrated diagnosis.
AP-04
CAP Synoptic Cancer Reporting
Form (WF-01 5.3) SPECIALIZED
Structured cancer checklist per CAP protocol template. Template selection: based on tumor type/site (e.g. 'Breast — Invasive Carcinoma, Excision' or 'Colon — Resection'). Fields per template: Procedure, Tumor Site, Tumor Size, Histologic Type (WHO classification), Histologic Grade, Margins (closest margin distance + status: negative/positive), Lymphovascular Invasion (Present/Absent), Perineural Invasion, Lymph Node Status (positive/total examined), TNM Staging (pT, pN, pM per AJCC 8th ed — auto-calculated stage group from T+N+M), Biomarkers (ER/PR/HER2 for breast, MSI/MMR for colon, etc.). Output formatted as synoptic report section within AP report.
ZVE AP SYNOPTIC wrapper -> structured synoptic data.
Template-driven: selecting tumor type loads the correct CAP protocol template with all required and conditional elements. AJCC stage group auto-calculated from T + N + M entries (e.g. pT2 + pN1 + pM0 = Stage IIB for breast). Required element validation: CAP mandates certain elements for accreditation — form prevents sign-out if required elements incomplete. Template versioning: CAP updates protocols quarterly — configuration must support version management.
AP-05
Report View / Sign-out
Detail View (WF-01 5.2)
Complete pathology report display. Header: Patient Name + DOB + MRN, Accession#, Specimen Description, Submitting Physician, Pathologist. Body sections (expandable): Clinical History, Gross Description (from AP-02), Microscopic Description (from AP-03), Diagnosis, Synoptic Report (from AP-04, if cancer case), Special Studies (IHC, molecular, etc.), Comments, Addenda. Signature block: Pathologist name + credential + date/time. Action bar: Sign/Verify (pathologist e-signature), Print, Addend (add supplemental report after sign-out).
ZVE AP REPORT wrapper -> LR AP report generation.
SIGN-OUT is IRREVERSIBLE (C/L3 per Audit Matrix): once signed, the pathology report becomes a permanent legal medical document. Corrections require Addendum (new entry linked to original — original preserved). Addenda common for: special stain results (received after initial sign-out), molecular testing results, outside consultation opinions. Report released to CPRS chart on sign-out — visible in Clinical workspace Lab Results section and Imaging section.
AP-06
Cytology Workstation
Form (WF-01 5.3) SPECIALIZED
Cytopathology-specific sign-out. GYN Cytology (Pap): Specimen Adequacy (Satisfactory/Unsatisfactory), Classification per Bethesda System (NILM, ASC-US, ASC-H, LSIL, HSIL, AGC, Adenocarcinoma, SCC), Recommendation (Routine screening / Repeat in 6 months / Colposcopy / etc.). Non-GYN Cytology (FNA, body fluids): Adequacy, Diagnosis, Recommendation. QC Queue: cytotechnologist screens slides, flags abnormals for pathologist review — QC dashboard shows: screened/pending/flagged counts.
ZVE AP CYTO wrapper -> LR AP cytology data.
Bethesda System dropdown: standardized classification for GYN cytology worldwide. QC workflow: 10% random re-screening required (CAP/CLIA). Abnormal cases flagged for pathologist review — cannot be released by cytotechnologist alone. HPV co-testing results integrated when available. Results link to Women's Health cervical screening tracking (Vol 17A — WH-03 integration).
AP-07
Autopsy Documentation
Form (WF-01 5.3) COMPLEX
Multi-section autopsy form: (1) Authorization (consent documentation, legal authority, restrictions), (2) External Examination (body habitus, identifying features, external injuries, clothing/personal effects), (3) Internal Examination (organ-by-organ: brain, heart, lungs, liver, kidneys, spleen, GI, reproductive — weight, gross findings per organ), (4) Histologic Examination (microscopic findings per organ/tissue sampled), (5) Ancillary Studies (toxicology, cultures, special stains), (6) Cause of Death (immediate cause, underlying cause, contributing conditions — per WHO death certification format), (7) Manner of Death (Natural/Accident/Suicide/Homicide/Undetermined/Pending). Provisional report + Final report (may be weeks later after ancillary results).
ZVE AP AUTOPSY wrapper -> LR AP autopsy data.
Two-phase reporting: Provisional report (within 24-48 hours of autopsy — external + internal gross findings + preliminary cause of death) and Final report (after all histology + ancillary studies complete — may be 30-60 days). Both reports signed separately. Organ weight table: auto-populated normal ranges by patient age/sex for comparison. Death certificate data: cause of death feeds to VistA PIMS for death certification.
AP-08
SNOMED / ICD-O Search
Detail View (WF-01 5.2)
Historical AP case search engine. Search by: SNOMED topography code (anatomic site), SNOMED morphology code (disease type), ICD-O-3 code, date range, submitting physician, pathologist, keywords in diagnosis text. Results: Accession#, Patient, Date, Specimen, Diagnosis preview. Click: opens full AP report.
ZVE AP SEARCH wrapper -> LR AP case index.
SNOMED 5-character progressive specificity: e.g. Respiratory 20000 -> Lung 28000 -> Right Upper Lobe 28200. Useful for: tumor board case retrieval ('show me all lung adenocarcinomas in the last 2 years'), quality reviews ('all amended reports this quarter'), research queries ('all renal biopsies with IgA nephropathy'). Exportable results list for tumor registry (ONC cross-link).
AP-09
AP Reports & Analytics
Report (WF-01 5.5)
Reports: Case volumes by sub-section (Surgical Path / Cyto / EM / Autopsy), turnaround times (accession to sign-out — target: routine <48h, rush <24h, frozen section <20 min), frozen section discordance rate (frozen diagnosis vs permanent diagnosis — quality metric), amended report rate (target: <2% — higher indicates potential accuracy issues), GYN cytology Bethesda distribution, pathologist workload (cases per day/month).
ZVE AP REPORTS wrapper -> LR AP data.
Standard Report template. Key quality metrics: turnaround time (CAP accreditation standard), frozen section concordance (measures intraoperative diagnostic accuracy), amended report rate (measures initial diagnostic accuracy). All reportable to CAP for accreditation.
AP-10
AP Configuration
Config (WF-01 5.6)
Configuration: accession area setup (SP, CY, EM, AU section codes), SNOMED code tables (national + local additions), CAP synoptic template management (import new versions when CAP releases quarterly updates), histology stain protocol definitions, report templates (header, footer, signature block format), autopsy report templates, workload profile definitions.
ZVE AP CONFIG wrapper -> LR AP configuration files.
CAUTION banner. CAP synoptic template updates: when CAP releases new protocol versions (quarterly), templates must be imported and validated before use. Template versioning: cases use the template version active at time of sign-out. SNOMED table maintenance: new codes added periodically — must be available in picker.
