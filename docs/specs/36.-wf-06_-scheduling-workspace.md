# VistA Evolved
Wireframe Specifications
WF-06: Scheduling Workspace
Document 6 of 16 | ~10 Pages | Appointment Scheduling, Clinic Management, Wait List, Recall, Check-In/Check-Out
March 2026 | The Scheduling workspace manages all outpatient appointment functions: creating and managing clinic availability (HOSPITAL LOCATION #44), scheduling patient appointments (APPOINTMENT sub-file of #44), check-in/check-out workflows, recall/reminder management, wait list tracking, and appointment reports. Used by: scheduling clerks, clinic staff, providers (for their own schedule view), and administrators (for capacity planning). VistA scheduling has historically been one of the most criticized components — VistA Evolved's scheduling must be a DRAMATIC improvement in usability. References Vol 2 (Scheduling) screen contracts.

SC-01
Appointment Calendar
Calendar (WF-01 5.7) PRIMARY INTERFACE
Calendar Header: View toggles (Day/Week/Month/Agenda), date navigation (< Today >), Clinic dropdown filter (multi-select), Provider dropdown filter, Appointment Type filter (All/Follow-Up/New Patient/Consult/Procedure/Telehealth). Calendar Grid: Day view shows time slots (configurable start/end per clinic, typically 07:00-17:00) as rows, selected clinics as columns. Week view: day columns, appointments as colored blocks. Month view: day cells with appointment count dots. Appointment blocks: Patient name, appointment type, duration, status color (Scheduled=blue, Checked-In=green, In-Exam=purple, Checked-Out=gray, No-Show=red, Cancelled=light gray strikethrough). Empty slot: click to create new appointment. Overbook slots: shown as additional small blocks overlapping regular slots (amber border indicating overbook).
ZVE SCHED CALENDAR wrapper -> HOSPITAL LOCATION #44 appointment sub-file.
DRAG-AND-DROP: drag patient from Patient Finder panel onto empty time slot to quick-schedule. Drag existing appointment to different time/date to reschedule (confirmation dialog). Conflict detection: double-booking same patient alerts, but allows with override. Overbook indicator: slots exceeding capacity shown with amber count badge ('2 over'). Telehealth appointments: video camera icon badge, with 'Start Video Visit' button appearing 15 minutes before appointment. Provider personal view: when provider is logged in, their schedule shown by default with patient details expanded.
SC-02
Appointment List / Search
Worklist (WF-01 5.1)
Searchable appointment list: Patient Name/MRN search, Clinic filter, Date range, Status filter (All/Scheduled/Checked-In/Completed/No-Show/Cancelled). Table: Date/Time, Patient, Clinic, Provider, Type (icon: Follow-Up/New/Consult/Procedure/Telehealth), Duration, Status badge, Check-In Time (if checked in), Check-Out Time (if completed), Wait Time (check-in to provider contact, calculated). Actions: Check In, Check Out, Cancel, Reschedule, No-Show, View Patient.
ZVE SCHED APPTS wrapper -> #44 appointment data.
Bulk actions: select multiple appointments for batch cancellation (e.g. provider out sick — cancel all their appointments for the day with single action + auto-notification to patients). Wait time tracking: time from check-in to first provider contact, displayed per appointment and aggregated per clinic for access reporting.
SC-03
Check-In / Check-Out
Form (WF-01 5.3) SPECIALIZED
CHECK-IN: Patient arrives, clerk selects appointment from today's schedule. Verification checklist: (1) Confirm patient identity (photo if available), (2) Verify/update demographics (address, phone, emergency contact — quick-confirm or edit), (3) Insurance verification prompt (if >30 days since last verified), (4) Copay notification (display copay amount due, collect or defer), (5) Travel pay eligibility prompt, (6) Health questionnaire completion check (pre-visit form). Check-in button timestamps arrival. CHECK-OUT: After visit, clerk selects patient. Check-out actions: (1) Schedule follow-up (launches SC-01 with patient pre-loaded), (2) Print after-visit summary (diagnoses, medications, instructions from today's visit), (3) Collect copay (if not collected at check-in). Check-out button timestamps departure.
ZVE SCHED CHECKIN wrapper -> #44 appointment status fields. IB copay calculation. Travel pay: BTSSS integration.
Check-in triggers: (1) appointment status changes to 'Checked-In' (green on calendar), (2) Nurse Dashboard notification (patient arrived), (3) wait time clock starts. Check-out triggers: (1) appointment status changes to 'Completed', (2) PCE encounter auto-created (Vol 4 Supplement integration), (3) wait time clock stops. Travel pay: eligible veterans can claim mileage reimbursement — integration with Beneficiary Travel Self-Service System (BTSSS).
SC-04
Clinic Profile Setup
Form (WF-01 5.3)
Clinic definition form: Clinic Name, Abbreviation, Service (Medicine/Surgery/Psychiatry/etc.), Provider(s) (multi-select), Location (building/floor/room), Phone, Operating Hours (start/end per day of week), Appointment Length (default minutes), Overbook Limit (max overbooks per slot), Slots Per Day (total capacity), Clinic Type (individual/group/procedure/telehealth), Pre-Appointment Instructions (text sent to patient), PCMM Team Link. Below: Slot Pattern grid — visual grid showing weekly pattern of available slots (e.g. MWF 08:00-12:00, Tu 13:00-17:00).
ZVE SCHED CLINIC wrapper -> HOSPITAL LOCATION #44 configuration sub-files.
Slot pattern grid: visual editor showing each day of week x time slot as a grid. Color slots: available (green), blocked (red), special (blue for reserved slots like new-patient-only). Copy pattern: duplicate one week's pattern to another. Holiday management: auto-blocks slots on federal holidays. Provider panel assignment: links clinic to PCMM team for primary care attribution.
SC-05
Clinic Availability View
Detail View (WF-01 5.2)
Calendar view showing availability across clinics: per-clinic: total slots, booked slots, available slots, overbooking count. Grid: clinics as rows, dates as columns, cell shows: available/total (color-coded green >50% available, amber 25-50%, red <25%). Useful for: finding next available appointment across multiple clinics, capacity planning. Drill-down: click cell to see specific open times.
ZVE SCHED AVAIL wrapper -> calculated from #44 slot data.
Scheduler's tool for finding appointment openings. 'Next Available' function: for a given clinic or set of clinics, find the earliest available slot matching criteria (appointment type, minimum duration, day-of-week preference). Result: list of available slots sorted by date, one-click to book.
SC-06
Electronic Wait List
Worklist (WF-01 5.1)
VHA Electronic Wait List per VHA Directive 1230. Table: Patient, Desired Date of Service, Clinic/Service Requested, Wait Status (Waiting/Scheduled/Removed), Date Added, Days Waiting (calculated), Priority (1 Emergent / 2 Urgent / 3 Routine), Comments. Actions: Schedule (opens SC-01 with patient pre-loaded), Remove from List (with reason: scheduled, no longer needs, patient request, etc.).
ZVE SCHED WAITLIST wrapper -> RECALL REMINDERS #403.5 or wait list file.
VHA MANDATE: veterans waiting >30 days for appointment must be tracked on electronic wait list. Days Waiting prominently displayed; >30 days = amber, >60 days = red. Automated notification to veteran when slot becomes available. Congressional reporting: wait list data feeds VSSC for congressional oversight of VA access. Wait time calculation: from 'desired date' to 'scheduled date'.
SC-07
Recall / Reminder Management
Worklist (WF-01 5.1) + Form
Patients due for follow-up: Patient, Last Visit Date, Recall Reason (e.g. 'Annual Physical', 'Diabetes Follow-Up 3mo', 'Post-Op 2 Weeks'), Recall Due Date, Outreach Status (Not Contacted / Letter Sent / Phone Attempted / Scheduled / Removed). Actions: Send Letter (auto-generates from template), Log Phone Attempt, Schedule (opens SC-01), Remove (with reason). New Recall Entry: Patient, Recall Date, Reason, Clinic/Provider.
ZVE SCHED RECALL wrapper -> RECALL REMINDERS #403.5.
Recall workflow: (1) provider creates recall at checkout ('return in 3 months'), (2) recall date calculated, (3) at configurable lead time (e.g. 2 weeks before due), auto-generates outreach letter or phone task, (4) scheduler contacts patient and schedules, (5) recall marked complete. Overdue recalls: patients past due date without scheduled appointment flagged for outreach.
SC-08
Scheduling Reports
Report (WF-01 5.5)
Reports: (1) Access Metrics: average wait time (from desired date to scheduled date, from check-in to provider), new patient wait time, next available appointment by clinic. (2) Utilization: slots used vs available, no-show rates, cancellation rates, overbooking frequency, provider productivity (patients per session). (3) Wait List: current wait list size, days waiting distribution, wait list outcomes. (4) Recall: recall compliance rates, overdue recalls. (5) Patient Satisfaction: linked to survey data if available.
ZVE SCHED REPORTS wrapper -> aggregated scheduling data.
Standard Report template. KEY VHA METRICS: third-next-available appointment (T3NA — standard access measure), average completed wait time, no-show rate (target <10%), same-day access rate. Reports feed VSSC and SAIL dashboards for facility performance scoring. Congressional interest: wait time data is politically sensitive and must be accurate.
SC-09
Scheduling Configuration
Config (WF-01 5.6)
Configuration: (1) Default appointment lengths by type, (2) Overbook policies per clinic type, (3) Auto-appointment creation rules (e.g. auto-create follow-up at discharge), (4) Letter/Reminder templates (appointment confirmation, recall, no-show), (5) Check-in kiosk configuration (VetLink/Vecna integration), (6) Telehealth settings (video visit platform integration), (7) Wait list parameters (auto-notification, aging thresholds), (8) Holiday schedule management.
ZVE SCHED CONFIG wrapper -> scheduling configuration files.
CAUTION banner. Check-in kiosk integration: VetLink/Vecna self-service kiosks allow veteran self-check-in with ID card scan (per Device Catalog). Telehealth integration: VA Video Connect or commercial platform link generation. Holiday schedule: federal holiday calendar auto-loaded, local holidays manually added.
SC-10
Provider Schedule View
Detail View (WF-01 5.2)
Provider-centric schedule view: MY day/week shown with: appointment times, patient names, appointment types, visit reasons (from consult or recall), patient preparation status (pre-visit labs done? questionnaires complete? prior records reviewed?). Huddle sheet: printable daily schedule with key patient information for morning team huddle. In-basket: messages from patients (My HealtheVet Secure Messaging), pending co-signatures, unsigned notes.
ZVE SCHED PROVIDER wrapper -> #44 appointments + patient data aggregation.
Provider's personal productivity view. Huddle sheet: one-page per-patient summary for each day's appointments (patient name, reason, key problems, meds, recent labs, reminders due, care gaps). Printable for morning team huddle. Pre-visit planning integration: flags patients who need lab work before visit, allowing pre-visit outreach.
