WF-07: Radiology / Imaging Workspace
Document 7 of 16 | ~12 Pages | Order Management, Exam Tracking, Report Dictation/Sign-out, DICOM Viewer Integration, Nuclear Medicine, Mammography, Imaging Reports
March 2026 | The Radiology/Imaging workspace manages the complete imaging lifecycle: order receipt and protocoling, exam scheduling, patient tracking (arrival, prep, exam-in-progress, complete), image acquisition via PACS/DICOM, report dictation and sign-out, result notification to ordering provider, and imaging quality metrics. Modalities: X-Ray, CT, MRI, Ultrasound, Nuclear Medicine, Mammography, Fluoroscopy, Interventional Radiology, PET/CT. VistA Radiology/Nuclear Medicine package (RA namespace) stores orders and reports; VistA Imaging (MAG namespace) stores and serves DICOM images via DICOMweb (WADO-RS/STOW-RS/QIDO-RS). References Vol 7 (Radiology/Imaging) screen contracts.

RA-01
Technologist Worklist
Worklist (WF-01 5.1)
The tech's primary screen for managing exams. Worklist Header: Modality filter (All/XR/CT/MR/US/NM/MG/FL/IR), Status filter (Ordered/Protocoled/Scheduled/Arrived/In-Progress/Complete/Cancelled), Date range, Location filter (room/scanner). Table columns: Patient (name + MRN + DOB + age + weight for contrast dosing), Exam (procedure name + CPT code), Modality icon (color-coded per modality type), Priority (STAT=red pulsing, Urgent=amber, Routine=gray, Scheduled=blue), Ordering Provider + Clinical Indication, Scheduled Date/Time, Status badge (color-coded lifecycle), Accession Number, Assigned Radiologist, Prep Requirements (NPO badge, IV contrast badge, sedation badge, pregnancy screen badge). STAT orders pinned to top. Auto-refresh every 30 seconds.
RAD/NUC MED ORDERS #75.1 via ZVE RAD WORKLIST wrapper.
Status lifecycle: Ordered -> Protocoled -> Scheduled -> Arrived -> In-Progress -> Complete -> Dictated -> Signed. Each transition has timestamp. Tech actions per status: 'Arrived' (patient is here), 'Begin Exam' (starts imaging), 'Complete Exam' (images sent to PACS), 'Cancel' (with reason). Prep requirement badges: critical for patient safety (NPO status for contrast studies, pregnancy screening for females of childbearing age, allergy to contrast pre-treatment protocol). Weight display: essential for CT contrast dosing calculations.
RA-02
Radiologist Reading Worklist
Worklist (WF-01 5.1)
Studies awaiting radiologist interpretation. Columns: Patient, Exam, Modality icon, Priority (STAT/Urgent/Routine), Study Date/Time, Referring Provider, Clinical History (truncated with hover-expand), Comparison Studies Available (count badge linking prior studies), Assigned Radiologist, Status (Complete awaiting read / In Dictation / Dictated awaiting sign-out), Age of Study (hours since completion — color-coded: <2h green, 2-8h amber, >8h red for STAT; longer thresholds for routine). Actions: Open in Viewer (launches RA-05 DICOM viewer), Dictate (opens RA-03), View Prior Reports. Sort: default by priority then study age (oldest STAT first).
RAD/NUC MED ORDERS #75.1 via ZVE RAD READLIST wrapper.
Radiologist's PRIMARY screen. Age-of-study tracking drives turnaround compliance. Comparison study auto-matching: system identifies prior studies of same modality and body region for the patient, displays count badge. One-click to open current + most recent prior in split viewer. STAT studies: turnaround target <1 hour from completion to preliminary report. Unread STAT studies >1 hour: red alert + notification to department chief.
RA-03
Report Dictation / Editing
Form (WF-01 5.3) SPECIALIZED
Split-screen: Left 55%: Report editor (rich text with structured sections: Clinical Indication, Technique, Comparison, Findings, Impression). Right 45%: Thumbnail image strip from current study (click to open in RA-05 viewer). Report templates: pre-loaded per exam type (e.g. CT Abdomen template has pre-populated section headers with standard normal findings that radiologist modifies). Voice dictation integration: speech-to-text via Dragon Medical / Nuance PowerScribe (or equivalent). Macro/Auto-text: abbreviation expansion (e.g. '.nl' expands to 'No acute abnormality identified.'). Structured reporting option: clickable structured fields for certain exams (e.g. BI-RADS for mammography, LI-RADS for liver, TI-RADS for thyroid, PI-RADS for prostate). Bottom: 'Sign Report' (e-signature — FINAL report), 'Save Draft', 'Preliminary Report' (communicates initial findings before final sign-out).
ZVE RAD REPORT wrapper -> TIU #8925 (Radiology report class) + RAD/NUC MED REPORTS #74.
Voice dictation: radiologist dictates while viewing images — speech-to-text fills report body. Template-driven: exam-type-specific templates reduce typing and ensure completeness. Preliminary report: for STAT studies, radiologist can issue preliminary findings immediately (communicated to ordering provider within minutes), followed by final report after complete review. Critical finding workflow: if radiologist identifies critical/unexpected finding (e.g. pulmonary embolism on routine chest CT), system requires: (1) direct communication to ordering provider (phone/secure message), (2) documentation of communication (who was notified, when, what was communicated) per ACR Practice Guideline.
RA-04
Order Entry / Protocoling
Form (WF-01 5.3)
Two modes: (A) ORDER ENTRY (ordering provider): Exam Type (searchable: 'CT Abdomen with Contrast'), Clinical Indication (required — ICD-10 + free text clinical question), Urgency (STAT/Urgent/Routine), Pregnancy Status for females (LMP date or pregnancy test result), Contrast Allergy check (auto from patient allergy list — flags iodinated contrast allergy with pre-treatment protocol suggestion), Transportation mode (ambulatory/wheelchair/stretcher/bed). (B) PROTOCOLING (radiologist/PA): Review order, assign specific protocol (e.g. CT Abdomen -> 'CT Abd/Pelvis with and without contrast, 3-phase liver protocol'), adjust contrast parameters (type, volume, rate), add special instructions, assign to scanner/room. Protocol approval routes order to scheduling.
ZVE RAD ORDER wrapper -> ORDER #100 + RAD/NUC MED ORDERS #75.1.
Ordering provider side: clinical decision support — appropriateness criteria display (ACR Appropriateness Criteria integration showing whether ordered exam is 'Usually Appropriate', 'May Be Appropriate', or 'Usually Not Appropriate' for the given indication). 'Usually Not Appropriate' triggers: recommendation for alternative study + override-with-reason if provider proceeds. Protocoling side: radiologist customizes the technical protocol based on clinical question. Scheduled exam: flows to technologist worklist (RA-01) at scheduled time.
RA-05
DICOM Image Viewer
Detail View (WF-01 5.2) SPECIALIZED — EXTERNAL INTEGRATION
Viewer launch page: study selector showing current study + prior studies for the patient. 'Open in Viewer' button launches the DICOM viewer. Viewer types: (A) Integrated DICOMweb viewer (embedded in browser via Cornerstone.js, OHIF Viewer, or equivalent open-source DICOM viewer — for web-based diagnostic viewing), (B) External PACS viewer launch (for sites using commercial PACS with proprietary viewer — launches via URL protocol handler). Viewer capabilities required: windowing (W/L), zoom, pan, scroll through series, measurements (linear, area, angle, Hounsfield density), annotations, comparison (side-by-side current vs prior), series linking (scroll-sync between series), 3D MPR/MIP reconstruction for CT/MRI, mammography-specific hanging protocols (CC/MLO layout).
VistA Imaging via DICOMweb (WADO-RS for image retrieval, QIDO-RS for query). ZVE RAD VIEWER wrapper for study metadata.
DIAGNOSTIC QUALITY: viewer must support radiologist diagnostic reading (not just clinical reference viewing). Requires: calibrated monitor support (DICOM GSDF grayscale standard display), full bit-depth rendering (12-16 bit for CT/MRI), mammography-specific display (5MP resolution for tomosynthesis). Hanging protocols: auto-layout of series based on exam type (e.g. CT Chest: axial soft tissue, axial lung, coronal, sagittal auto-arranged). Prior study comparison: auto-fetches most recent relevant prior from VistA Imaging. Key-image flagging: radiologist marks key images, which display as thumbnails in the radiology report.
RA-06
Mammography / BI-RADS Tracking
Worklist (WF-01 5.1) + Form SPECIALIZED
Mammography-specific workflow per MQSA (Mammography Quality Standards Act). Worklist: all mammography exams by patient with BI-RADS tracking. Columns: Patient, Exam Date, Exam Type (Screening/Diagnostic), BI-RADS Assessment (0-6 with color-coded badges: 0=amber 'Needs Additional Imaging', 1-2=green 'Negative/Benign', 3=yellow 'Probably Benign — Short Interval Follow-Up', 4=orange 'Suspicious', 5=red 'Highly Suggestive of Malignancy', 6=red 'Known Malignancy'), Recommendation, Follow-Up Status (letter sent, callback complete, biopsy scheduled), Days Since Exam. BI-RADS entry form: structured fields per ACR BI-RADS lexicon (breast density: A-D, mass descriptors, calcification descriptors, associated features, assessment category, recommendation).
RAD/NUC MED mammography data via ZVE RAD MAMMO wrapper. Women's Health integration: ZVE WH BREAST wrapper (Vol 17A WH-02).
MQSA COMPLIANCE: FDA-regulated program. Requirements enforced: (1) every mammogram must have BI-RADS assessment documented, (2) patient notification letter must be sent within 30 days (lay-language results per MQSA), (3) referring provider notification of results, (4) abnormal result follow-up tracking (BI-RADS 0, 4, 5 must have documented follow-up — biopsy recommendation, additional imaging). Women's Health integration: BI-RADS results auto-populate WH-02 breast screening tracking (Vol 17A). Lead Interpreting Physician (LIP) designation tracked per MQSA.
RA-07
Mammography Patient Letter Generation
Form (WF-01 5.3)
MQSA-required patient notification letter. Template-driven: letter text auto-generated from BI-RADS assessment + exam date + recommendation. Language: lay-person understandable per FDA MQSA requirements (no medical jargon — e.g. 'Your mammogram showed no signs of cancer' for BI-RADS 1). Letter includes: facility contact information, recommendation for next screening date (BI-RADS 1-2) or follow-up instructions (BI-RADS 0, 3, 4, 5). Actions: Generate Letter, Preview, Print, Mail (marks as sent). Batch generation: generate all pending letters for a date range.
ZVE RAD LETTER wrapper.
Letter templates per BI-RADS category: separate template for each assessment category. Multi-language support: per I18n Framework (Phase 2), letters generated in patient's preferred language. Tracking: system logs letter generation date, mailing date, and whether acknowledgement received. Overdue letters (>30 days from exam without letter sent): RED alert on mammography worklist.
RA-08
Nuclear Medicine Management
Worklist (WF-01 5.1) + Form
NM-specific worklist: radiopharmaceutical dose tracking. Columns: Patient, Exam, Radiopharmaceutical (agent name + activity in mCi/MBq), Dose Prep Time, Injection Time, Scan Time, Status. Dose preparation form: agent from NM formulary, activity measured (dose calibrator reading), calibration time, lot#, expiration. Regulatory: NRC (Nuclear Regulatory Commission) compliance — maximum permissible dose per administration, misadministration reporting threshold. Radiation safety: pregnancy verification MANDATORY before any NM exam.
ZVE RAD NM wrapper -> RAD/NUC MED NM-specific files.
Radiopharmaceutical dose decay correction: auto-calculates activity at injection time based on preparation time and isotope half-life (e.g. Tc-99m T1/2 = 6.02 hours). Dose threshold monitoring: if administered activity deviates >20% from prescribed dose, system flags as potential misadministration requiring NRC reporting. Wipe test / survey results: radiation safety documentation per NRC license conditions.
RA-09
Interventional Radiology Procedure Log
Form (WF-01 5.3) + Detail View
IR procedure documentation: pre-procedure (consent, time-out verification, site marking), intra-procedure (access, devices used, contrast volume, fluoroscopy time, radiation dose DAP/kerma), post-procedure (outcomes, complications, disposition). Device tracking: catheters, stents, embolization materials — lot#, model#, manufacturer for recall tracking. Radiation dose recording: cumulative patient dose per exam (dose reference levels, skin dose estimates). Links to Surgery workspace for hybrid OR procedures.
ZVE RAD IR wrapper -> RAD/NUC MED + procedure-specific data.
Radiation dose tracking: mandatory per Joint Commission and ACR. Cumulative dose alerts: if patient's cumulative radiation exposure exceeds threshold (e.g. >100 mSv in 12 months), system alerts radiologist to consider alternative non-radiation imaging. Fluoroscopy time recording: real-time from fluoroscopy unit (if digitally integrated) or manual entry. Complication tracking for quality reporting: vascular access complications, contrast reactions, post-procedure infections.
RA-10
Imaging Reports & Quality Metrics
Report (WF-01 5.5)
Reports: (1) Turnaround Times: order-to-report by modality and priority (STAT target: <1h for preliminary, <24h for final; Routine target: <48h for final). (2) Volume: exams by modality, by date, by radiologist. (3) Critical Result Communication: compliance with ACR guideline (documented direct communication for unexpected critical findings). (4) Repeat/Reject Analysis: images rejected for quality (motion, positioning, exposure) — rate and reasons. (5) MQSA: mammography volume per radiologist (minimum 480/year for LIP), BI-RADS distribution, cancer detection rate, recall rate, positive predictive value. (6) Radiation Dose: facility dose indices vs national diagnostic reference levels (DRLs). (7) Peer Review: RADPEER scoring (1-4 discordance scale) for quality assurance.
ZVE RAD REPORTS wrapper -> aggregated imaging data.
Standard Report template. MQSA reports: FDA-auditable — must be accurate for accreditation. RADPEER: anonymous peer review system where radiologists retrospectively score peers' prior interpretations when viewing comparison studies (score 1=agree, 2=discrepancy not clinically significant, 3=discrepancy — should have been made, 4=discrepancy — definitely should have been made). Radiation dose reporting: ACR Dose Index Registry (DIR) compatible export.
RA-11
Exam Type / Protocol Configuration
Config (WF-01 5.6)
Configuration: (1) Exam Types: procedure names, CPT codes, modality assignment, default protocol, prep requirements, estimated duration, contrast requirements. (2) Report Templates: per exam type, section headers, standard normal templates, structured reporting field definitions (BI-RADS, LI-RADS, TI-RADS, PI-RADS, Lung-RADS). (3) Protocol Library: per exam type, scanner-specific protocols (slice thickness, contrast timing, sequences for MRI). (4) Hanging Protocol definitions: how series are arranged on the viewer per exam type.
ZVE RAD CONFIG wrapper -> RAD/NUC MED configuration files.
CAUTION banner. Exam type changes affect: ordering interface, worklist display, protocoling options, report templates, billing codes. Protocol library: scanner-vendor-specific (GE, Siemens, Philips protocols differ). Structured reporting templates: must match current ACR/MQSA standards, updated when criteria revisions are published.
RA-12
PACS / VistA Imaging Connection Management
Config (WF-01 5.6)
PACS integration dashboard: connection status to PACS nodes (VistA Imaging DICOM Gateway, commercial PACS if applicable), DICOM modality worklist (MWL) server status, storage commitment status, image routing rules (which modalities route to which storage). DICOM node table: AE Title, IP Address, Port, Status (online/offline), Last Communication, Throughput. Image lifecycle: how long images stored online vs nearline vs archive. Prefetch rules: auto-retrieve relevant prior studies when new order is created.
ZVE RAD PACS wrapper -> VistA Imaging MAG configuration, DICOM Gateway settings.
CAUTION banner — technical configuration. DICOM connectivity: C-STORE (send images), C-FIND (query), C-MOVE (retrieve), WADO-RS/STOW-RS/QIDO-RS (web access). Modality Worklist (MWL): provides patient/order data to imaging modality so technologist does not re-enter demographics at the scanner (prevents wrong-patient image filing). Image routing: rules defining where images are sent after acquisition (e.g. CT scanner -> PACS archive -> VistA Imaging for clinical viewing). Storage management: online (SSD for recent studies), nearline (spinning disk), archive (tape/cloud for long-term per retention policy — typically 7 years minimum, mammography 10 years per MQSA).
