WF-08: Billing / Revenue Workspace
Document 8 of 16 | ~24 Pages | Revenue Dashboard, Charge Capture, Claims Management, Payments, Accounts Receivable, Fee Basis/Community Care, Rate Management, Copay Billing
March 2026 | The Billing/Revenue workspace manages the complete revenue cycle: charge capture from clinical encounters (IB namespace), claims generation and submission (IB + third-party payer interface), payment posting, accounts receivable management (AR/PRCA namespace), first-party (veteran copay) billing, third-party (insurance) billing, Fee Basis/Community Care payment processing, rate management, and revenue analytics. VA billing is unique: dual-payer model (VA is both payer and provider), with third-party collections authorized under the Affordable Care Act provisions and MCCF (Medical Care Cost Fund). References Vol 8 (Billing/Revenue) + Vol 8 Supplement (AR) screen contracts.

BL-01
Revenue Cycle Dashboard
Dashboard (WF-01 5.4)
KPI card row (5 cards): Total Charges (period, trend arrow), Claims Submitted (count + dollar), Collections (third-party + first-party), Rejection Rate (% + trend), AR Aging (30/60/90/120+ day bars in mini bar chart). Below KPIs: two-panel layout. Left panel: Claims Pipeline funnel chart (Unbilled -> Coded -> Submitted -> Adjudicated -> Paid/Denied). Right panel: AR Aging stacked bar chart (current + 30 + 60 + 90 + 120+ day buckets by payer category). Bottom: Action items worklist (rejected claims needing attention, unbilled encounters aging, AR accounts needing follow-up).
ZVE BILLING DASHBOARD wrapper -> aggregated IB + AR data.
Executive overview for Revenue Cycle Manager. Drill-down: click any KPI card or chart segment to navigate to filtered worklist. Trend arrows compare current period to prior period. Period selector: Today/Week/Month/Quarter/Year/Custom. Division filter for multi-division facilities. Auto-refresh every 5 minutes.
BL-02
Unbilled Encounter Worklist
Worklist (WF-01 5.1)
Encounters completed but not yet billed. Columns: Patient, Encounter Date, Clinic, Provider, Encounter Type (outpatient/inpatient), Diagnoses (ICD-10 list), Procedures (CPT list), Insurance Status (verified/unverified/none), Days Unbilled (amber >7d, red >14d), Coding Status (Not Coded/Coded/Ready to Bill). Actions: Code (opens BL-03), Submit (generates claim), Hold (with reason), Flag for Review.
IB encounter data via ZVE BILLING UNBILLED wrapper.
Revenue leakage prevention: every clinical encounter should generate a bill. Unbilled encounters aging report: encounters >14 days without claim submission flagged as revenue at risk. Auto-coding: for encounters with documented CPT and ICD-10 from PCE (Vol 4 Supplement), system auto-populates coding fields. Manual coding: for complex encounters or missing codes, coder reviews chart and assigns.
BL-03
Charge Entry / Coding
Form (WF-01 5.3)
Coding workstation: Left 60%: encounter detail (patient, date, provider, clinical notes summary). Center: Diagnosis entry (ICD-10 searchable, primary + additional — up to 12), Procedure entry (CPT/HCPCS searchable with modifier dropdowns — 25, 26, 59, TC, etc.), Quantity, Place of Service, Rendering Provider. Right 40%: Coding assistance panel — encounter notes with AI-suggested codes highlighted, NCCI edit checker (National Correct Coding Initiative — flags bundling violations), LCD/NCD coverage determination check (is this CPT covered for this ICD-10?). Bottom: Charge amount (auto-calculated from Rate tables BL-18/BL-19), Submit button.
ZVE BILLING CODE wrapper -> IB #350 (billing file). CPT: CPT #81. ICD: ICD DIAGNOSIS #80.
NCCI edit checking: real-time validation that CPT code combinations comply with CMS NCCI rules (e.g. certain procedures cannot be billed together, or require modifier 59 for separate procedure). LCD/NCD check: verifies medical necessity — is the procedure covered by Medicare/payer for the given diagnosis? Fails: warns coder, suggests alternative diagnosis or notes medical necessity documentation needed. Revenue impact: accurate coding maximizes appropriate reimbursement; under-coding loses revenue, over-coding risks audit/compliance.
BL-04
Inpatient Coding / DRG Assignment
Form (WF-01 5.3)
Inpatient episode coding: Patient, Admission/Discharge dates, LOS, Principal Diagnosis (ICD-10-CM), Secondary Diagnoses (up to 24), Principal Procedure (ICD-10-PCS), Additional Procedures, DRG Assignment (auto-calculated from diagnoses + procedures using MS-DRG grouper), DRG Weight, Expected Reimbursement. CC/MCC indicator: whether secondary diagnoses qualify as Complications/Major Complications (which shift to higher-paying DRG). Query panel: physician query generator when documentation is insufficient for optimal coding.
ZVE BILLING DRG wrapper -> PTF #45 + IB #399. DRG grouper: 3M or equivalent.
DRG assignment drives inpatient reimbursement. CC/MCC analysis: system identifies secondary diagnoses that qualify as complications (increasing DRG weight). CDI (Clinical Documentation Improvement): if documentation supports a CC/MCC but coding is missing it, system highlights the opportunity. Physician query: structured query to attending physician requesting clarification (e.g. 'Does the patient have acute respiratory failure? Documentation suggests but does not explicitly state.').
BL-05
Third-Party Claims Worklist
Worklist (WF-01 5.1)
Claims tracking: Claim ID, Patient, Payer, Service Date, Amount Billed, Status (Generated/Submitted/Accepted/Adjudicated/Paid/Denied/Appealed), Paid Amount, Adjustment Amount, Days Since Submitted. Filter: Status, Payer, Date, Amount range. Sort by aging (oldest unresolved first). Split view: right panel shows claim detail on selection.
IB CLAIMS data via ZVE BILLING CLAIMS wrapper.
Claim lifecycle tracking: Generated (coded, not yet sent) -> Submitted (transmitted to payer via 837P/837I EDI) -> Accepted (payer acknowledged receipt) -> Adjudicated (payer processed) -> Paid (remittance received) or Denied (with denial reason code). Denied claims: auto-categorized by denial reason for targeted resolution workflow.
BL-06
Claim Submission / 837 Generation
Form (WF-01 5.3) + Worklist
Batch claim submission: select claims from Generated status, validate (scrub for errors before submission), submit as 837P (professional) or 837I (institutional) EDI file to clearinghouse or payer. Pre-submission scrub: checks for: missing demographics, invalid codes, payer-specific requirements, timely filing deadline approaching. Error list: claims failing scrub shown with specific errors + fix instructions. Individual claim edit: correct specific fields before resubmission.
ZVE BILLING SUBMIT wrapper -> 837 EDI generation.
Clearinghouse integration: 837 files transmitted to clearinghouse (or direct to payer) per HIPAA X12 format. Pre-submission scrub catches errors BEFORE submission (reducing rejections). Timely filing: displays days remaining before payer's filing deadline (typically 90-365 days depending on payer). Claims approaching deadline: red highlight for urgent submission.
BL-07
Denial / Rejection Management
Worklist (WF-01 5.1) + Form
Denied claims needing action: Claim ID, Patient, Payer, Denial Reason Code + Description (CARC/RARC codes translated to human-readable), Denial Category (Eligibility, Authorization, Coding, Medical Necessity, Timely Filing, Duplicate, Other), Amount, Days Since Denial, Resolution Status. Actions: Correct and Resubmit, Appeal (opens BL-08), Write Off (with approval), Adjust.
ZVE BILLING DENY wrapper -> IB denial tracking.
Denial categorization auto-sorts for efficient workflow: eligibility denials -> check insurance (PT-06 link), authorization denials -> check auth status, coding denials -> review codes (BL-03 link), medical necessity -> add documentation. Appeal eligibility: system checks if denial is appealable and within appeal timeframe. Denial trending: reports show which denial reasons are most frequent to drive process improvement.
BL-08
Appeal Entry
Form (WF-01 5.3)
Appeal form: Original Claim reference, Denial Reason, Appeal Level (First-Level/Second-Level/External Review), Appeal Rationale (template-driven with denial-specific suggested language), Supporting Documentation (attach clinical records, operative reports, medical necessity letters). Submit to payer. Track appeal status and outcome.
ZVE BILLING APPEAL wrapper.
Multi-level appeal: First level (payer internal), Second level (payer internal, different reviewer), External review (independent review entity). Supporting documentation: system auto-identifies relevant clinical documents (procedure notes, lab results, imaging reports) based on claim CPT/ICD codes — coder selects which to include. Appeal success rate tracking: by payer and denial reason.
BL-09
Payment Posting (ERA/EOB)
Form (WF-01 5.3) + Worklist
Electronic Remittance Advice (835 ERA) processing: import ERA file, auto-match payments to claims, display: Claim ID, Billed Amount, Allowed Amount, Paid Amount, Adjustment Amount + Reason (contractual, deductible, coinsurance), Patient Responsibility. Actions: Accept (posts payment), Adjust (manual correction), Deny Match (ERA line does not match any claim — investigation needed). Batch posting: accept all matched payments in single action.
ZVE BILLING PAYMENT wrapper -> IB + AR payment posting. 835 ERA import.
Auto-matching: 835 ERA lines matched to claims by claim ID, patient, service date. Match confidence: green (exact match), amber (probable match — slight discrepancy, manual review), red (no match found). Contractual adjustment auto-posting: allowed amount vs billed amount difference auto-posted as contractual write-off per payer contract. Patient responsibility: copay/deductible/coinsurance amounts flow to AR for patient billing (BL-12).
BL-10
Manual Payment Entry
Form (WF-01 5.3)
For payments not received electronically (paper checks, cash, credit card): Patient/Payer, Amount, Payment Method (check#/EFT/cash/CC), Receipt Number, Claim Reference(s), Allocation (how payment is distributed across claims/AR balances). Receipt generation. Daily deposit reconciliation.
ZVE BILLING MANUAL PAY wrapper.
Multi-claim allocation: single payment may cover multiple claims. Over/under payment handling: overpayment creates credit balance (refund or apply to future), underpayment leaves remaining balance in AR. Daily deposit: sum of all payments received must reconcile with bank deposit.
BL-11
Payment Reconciliation
Dashboard (WF-01 5.4)
Daily/weekly/monthly reconciliation: Expected vs Received by payer. Columns: Payer, Claims Submitted (count + $), Payments Received ($), Adjustments ($), Outstanding ($), Collection Rate (%). Discrepancy highlighting: payers with collection rate below threshold (amber <90%, red <80%).
ZVE BILLING RECON wrapper -> IB + AR aggregate data.
Financial control: reconciliation ensures all expected payments are received. Underpayment identification: contracted rate vs actual payment comparison. Payer performance: identifies slow-paying or underpaying payers for contract negotiation.
BL-12
Patient Statements (First-Party Billing)
Worklist (WF-01 5.1) + Form
Veteran copay billing: outstanding patient balances. Columns: Patient, Balance Due, Last Statement Date, Statement Cycle (30/60/90), Payment Plan (Y/N + terms), Hardship Waiver (Pending/Approved/Denied). Actions: Generate Statement, Set Up Payment Plan, Initiate Hardship Review, Write Off (with approval per AR authority levels), Refer to Collections (Treasury/TOP). Statement preview: formatted bill showing charges, payments, balance.
ACCOUNTS RECEIVABLE #340 + IB via ZVE AR PATIENT wrapper.
VA first-party billing: veterans may owe copays for non-service-connected care based on Priority Group and Means Test. Statement cycles: 30-day billing cycle with escalating collection letters. Hardship waiver: for veterans unable to pay, waiver process (per VHA Directive) considers income, assets, and circumstances. Treasury Offset Program (TOP): delinquent debts >120 days may be referred to US Treasury for collection from tax refunds/federal payments.
BL-13
AR Follow-Up Worklist
Worklist (WF-01 5.1)
Third-party AR aging: accounts needing follow-up. Columns: Payer, Account/Claim ID, Patient, Amount Owed, Days Outstanding (color-coded aging), Last Follow-Up Date, Next Action Date, Follow-Up Notes. Actions: Call Payer (log call), Re-submit Claim, Send Additional Documentation, Appeal, Write Off. Sort by amount (high-value first) or age (oldest first).
ACCOUNTS RECEIVABLE #340 via ZVE AR FOLLOWUP wrapper.
AR management: active follow-up on outstanding balances. Workflow: initial claim submission -> 30 days: if no response, re-inquiry -> 60 days: escalated follow-up with additional documentation -> 90 days: appeal or write-off decision -> 120+ days: refer to supervisor for action. Follow-up notes: chronological documentation of all collection activities for audit trail.
BL-14
AR Write-Off Processing
Form (WF-01 5.3)
Write-off form: Account reference, Amount to Write Off, Write-Off Category (Uncollectible, Administrative, Compromise, Court Order), Approval Authority (tiered by amount: <$500 supervisor, <$5000 manager, <$25000 director, >$25000 VISN/national), Justification narrative, Supporting documentation. Approval workflow: submitted for review, approved/denied.
ZVE AR WRITEOFF wrapper -> #340 write-off processing.
TIERED APPROVAL: write-off authority levels per VHA policy (higher amounts require higher authority). Compromise offers: veteran proposes partial payment in full settlement — requires evaluation and approval. All write-offs are IRREVERSIBLE (I/L3 per Audit Matrix) and audit-logged. Aggregate write-off reporting for financial statements.
BL-15
Fee Basis / Community Care Authorizations
Worklist (WF-01 5.1) + Form
Community Care authorizations: Patient, Authorized Service, External Provider, Auth#, Date Range, Budget Category, Amount Authorized, Amount Used, Status (Active/Expired/Cancelled). New Authorization form: Patient, Service (CPT/HCPCS), Provider/Facility (NPI search), Clinical Justification, Number of Visits or Episode, Date Range, Estimated Cost.
ZVE FEE AUTH wrapper -> FEE BASIS #161 + Community Care files.
Community Care (formerly Fee Basis): VA-authorized care at non-VA facilities. Authorization required BEFORE care is rendered (except emergencies). MISSION Act/COMPACT Act rules: eligibility criteria for community care (wait time, drive time, service unavailability). Budget tracking: authorized amounts vs actual claims from external providers.
BL-16
Fee Basis / CC Payment Processing
Worklist (WF-01 5.1) + Form
Claims from external providers: Invoice/Claim, External Provider, Patient, Service Dates, CPT/HCPCS codes, Billed Amount, Allowed Amount (per VA fee schedule), Amount to Pay. Matching: claim matched to authorization — if no auth, flagged for review. Actions: Approve Payment, Reduce to Allowed Amount, Deny (with reason), Request Additional Information.
ZVE FEE PAYMENT wrapper -> FEE BASIS payment files.
External provider claims adjudicated against VA fee schedules. Over-charge identification: billed amount exceeds VA allowed amount — auto-reduced. No authorization match: claim flagged — may be emergency (allow) or unauthorized (deny). Payment cycle: approved claims batched for payment via VA financial system (FMS/IFCAP integration).
BL-17
CC Referral Coordination
Detail View (WF-01 5.2)
Community Care referral tracking: referral source (consult from Clinical workspace), authorized services, external provider assignment, appointment status (scheduled/completed/no-show), clinical results received (Y/N), follow-up care coordination. Timeline view: referral created -> auth issued -> appointment scheduled -> care rendered -> results received -> follow-up at VA.
ZVE CC REFERRAL wrapper -> CC tracking data.
End-to-end community care coordination. Key pain point: ensuring clinical results from external providers are received back at VA and entered into the patient record. Results not received after configurable days: auto-reminder to CC coordinator for outreach. Patient navigation: CC coordinator as point of contact for veteran throughout community care episode.
BL-18
Rate Table Management
Detail View (WF-01 5.2) + Form
CPT/HCPCS rate tables: Code, Description, VA Charge Amount, Medicare Rate (for reference), Facility Component, Professional Component, Effective Date Range. Import: bulk rate table import from annual updates (Medicare MPFS, APC). Edit: individual rate adjustment. Multiple rate tables: First Party (copay calculation), Third Party (insurance billing), Fee Basis (community care payment).
ZVE RATE TABLE wrapper -> IB rate configuration.
Annual updates: Medicare physician fee schedule (MPFS) and hospital outpatient prospective payment (OPPS/APC) rates updated annually by CMS. Import utility: loads new rates from CMS data files. Effective dating: new rates activate on effective date while old rates remain for historical claims. VA-specific rates: may differ from Medicare for certain services.
BL-19
DRG / Inpatient Rate Management
Detail View (WF-01 5.2) + Form
MS-DRG weight table: DRG Code, Description, Weight, Geometric Mean LOS, Arithmetic Mean LOS, Base Rate (facility-specific), Expected Payment. Import: annual DRG update from CMS. Per diem rates: for non-DRG inpatient billing (certain payers, long-term care). Rate comparison: VA rate vs Medicare rate vs commercial payer negotiated rates.
ZVE RATE DRG wrapper -> IB DRG configuration.
DRG-based inpatient billing: payment = base rate x DRG weight. Outlier calculations: very high-cost cases may qualify for additional payment beyond DRG. CMS annual update: new DRGs, revised weights, reclassified codes — import and validate before effective date.
BL-20
Billing Reports
Report (WF-01 5.5)
Reports: Revenue summary (charges, collections, adjustments by period), Collection rates by payer, Denial analysis (top denial reasons, denial rates by payer, appeal success rates), AR aging analysis (30/60/90/120+ by payer category), Unbilled encounter aging, Coding productivity (encounters coded per coder per day), Community Care spending (authorizations vs payments), MCCF (Medical Care Cost Fund) revenue tracking, Compliance reports (coding accuracy, documentation completeness).
ZVE BILLING REPORTS wrapper -> aggregated IB + AR + Fee data.
Standard Report template. MCCF tracking: third-party collections revenue reported separately (feeds VHA budget). Compliance reports: coding accuracy sampled audits, documentation completeness rates. Community Care budget: authorized vs actual spending for budget management.
BL-21 to BL-24
Copay Configuration, Payer Setup, Claim Rules Engine, Master Billing Config
Config (WF-01 5.6)
BL-21 (Copay Config): first-party copay rates per service category (outpatient visit, specialty visit, pharmacy, inpatient per diem) per Priority Group. BL-22 (Payer Setup): third-party payer profiles (name, payer ID, EDI address, timely filing limit, contract terms, fee schedule link). BL-23 (Claim Rules): auto-billing rules (which encounters auto-generate claims), scrub rules (pre-submission validation), denial management rules. BL-24 (Master Config): billing division setup, clearinghouse connection, statement cycle parameters, write-off authority tiers, Treasury offset parameters.
Various ZVE BILLING CONFIG wrappers.
CAUTION banner on all config pages. Copay rates set by federal regulation (updated annually by Congress/VA). Payer setup critical: incorrect payer ID causes all claims to that payer to reject. Claim rules engine: configurable business rules automating billing workflow. Changes require Revenue Cycle Manager approval.
