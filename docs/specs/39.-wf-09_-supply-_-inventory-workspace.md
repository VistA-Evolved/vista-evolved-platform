# VistA Evolved
Wireframe Specifications
WF-09: Supply / Inventory Workspace
Document 9 of 16 | ~10 Pages | Supply Dashboard, Inventory Catalog, Requisitions, Purchase Orders, Receiving, Distribution, Equipment Tracking, Physical Inventory, Reports, Configuration
March 2026 | The Supply/Inventory workspace manages the complete supply chain for healthcare facility operations: inventory control of consumable supplies and durable goods, purchasing via the Integrated Funds Distribution, Control Point Activity, Accounting and Procurement system (IFCAP), receiving and warehousing, distribution to clinical areas, capital and non-capital equipment lifecycle tracking, physical inventory cycle counting, and expenditure reporting. VistA packages: Generic Inventory Package (GIP) for inventory management, IFCAP for procurement and financial accountability, Engineering (AEMS/MERS) for equipment management. This workspace is used by: supply chain managers, purchasing agents, warehouse staff, receiving clerks, control point officials (budget approvers), biomedical engineering coordinators, and unit supply coordinators on nursing wards. References Vol 9 (Supply/Inventory) screen contracts.

# 1. Supply/Inventory Workspace Navigation Structure
The Supply workspace uses HORIZONTAL TAB BAR (WF-01 Sub-Navigation Pattern 1) with 7 tabs. No patient selection required for this workspace — it operates on facility-level inventory data, not patient-specific data. The Patient Banner zone is replaced by a Location Context Bar showing the currently selected warehouse/distribution point.

This prevents over-commitment of funds. Emergency bypass: for true emergencies (e.g. critical medical supply needed for patient care), requisition can be submitted even if over-budget — generates special approval workflow to facility director.

Amendment audit trail: every PO change documented with before/after values, reason, and approver (changes above certain $ thresholds require re-approval). Contract compliance: for items on VA national contracts, system verifies that order uses contract vendor and contract pricing — deviation requires justification. Blanket Purchase Agreements (BPA): for recurring items, BPAs allow multiple delivery orders against a single contract — tracked as BPA releases under parent BPA.

Color coding per row: green (received qty matches expected), amber (partial receipt — less than expected), red (damaged or wrong item).
Layout — Discrepancy Panel (bottom, expandable)
If any line has condition other than 'Good' or quantity mismatch: discrepancy documentation section expands. Per discrepancy: Line item, Discrepancy Type (Short Shipment / Over Shipment / Damage / Wrong Item / Substitution / Quality Failure), Description (free text with photos — device camera capture), Resolution (Accept As-Is / Return to Vendor / Credit Memo / Reorder / Reject). Generate vendor notification: auto-email/letter to vendor documenting discrepancy and requesting resolution. Discrepancy aging: unresolved discrepancies tracked on SU-01 Dashboard alerts.
Layout — Summary & Actions (bottom)
Receipt summary: Total Lines, Lines Complete, Lines with Discrepancies. Actions: 'Complete Receiving' (posts all lines to inventory — updates GIP quantities in SU-02), 'Save Partial' (some lines received, more expected later), 'Print Receiving Report' (formatted report for file and 3-way match documentation). On Complete: inventory quantities updated immediately, items appear at assigned storage locations in SU-02, PO status advances (Partially Received if some lines remain, Fully Received if all complete), IFCAP expenditure posted (fund control point balance reduced by actual expenditure).
Data Source
ZVE SUPPLY RECEIVE wrapper -> GIP (inventory adjustment), IFCAP #442 (PO receiving data), IFCAP financial posting (expenditure). Lot/Expiration: stored in GIP item lot tracking sub-files.
Unique Behaviors
Barcode scan receiving: scan each item's barcode/UPC as it is unpacked — system matches scanned item to PO line and auto-populates the row (prevents receiving the wrong item). Visual confirmation: scanned item name displayed prominently for clerk to visually verify against physical item. Temperature-sensitive items: for cold-chain items (vaccines, certain lab reagents), receiving form includes temperature logger reading field (was the cold chain maintained during shipping?). 3-way match completion: when both receiving report AND vendor invoice are in system, auto-comparison triggers. Match = auto-approve for payment. Mismatch (price difference, quantity difference) = flag for purchasing agent resolution before payment. FDA UDI: for medical devices, Unique Device Identifier scanned and recorded for recall traceability (links to device manufacturer's GUDID database). Recall check on receiving: system checks received item lot numbers against active recall list — if match, BLOCKS acceptance and alerts supply chain manager.

Data Source

Warehouse pick routing: pick list sorted by warehouse aisle/shelf/bin for efficient walking path (reduces pick time for warehouse staff). Linen exchange: specialized distribution category where issued linen quantity must match returned soiled linen quantity (1-for-1 exchange prevents linen hoarding). Equipment checkout: for reusable shared equipment (infusion pumps, wheelchairs, PCA pumps), 'Issue' creates a checkout record with: item serial#, destination, checkout date, expected return date. Equipment not returned by expected date generates alert. Integration with Pharmacy Ward Stock (Vol 5F WS): pharmacy supplies issued through Pharmacy Ward Stock system (WS-01), not through general Supply — prevents duplicate inventory management. Supply workspace manages everything EXCEPT medications.

## SU-08: Physical Inventory / Cycle Counting
Aspect
Specification
Page ID
SU-08
Template
Form (WF-01 5.3) + Worklist (WF-01 5.1)
Layout — Count Planning (top section)
Cycle count schedule: calendar view showing which item categories are counted in each period (weekly/monthly/quarterly rotation). Current period highlighted. Categories: Class A (high-value, high-movement — counted monthly), Class B (moderate — counted quarterly), Class C (low-value, low-movement — counted annually). 'Generate Count Sheet' button: select location + category -> produces count document (item list with expected quantities blanked out for blind count).
Layout — Count Entry (main section)
Two modes: (A) Paper Count: print count sheets, physical count recorded on paper, then entered into system (Item, Expected Qty displayed AFTER count entered to prevent bias, Counted Qty entered by user, Variance auto-calculated). (B) Mobile Count: using mobile device or handheld scanner, scan item barcodes and enter counted quantities in real-time — system auto-matches scanned items to master inventory. Per-item count entry: Item Name, Stock#, Location, Unit of Issue, Counted Quantity (numeric entry), Expected Quantity (system value — hidden during count for blind counting, revealed after entry for comparison), Variance (counted minus expected — green if zero, amber if within tolerance, red if exceeds tolerance). Variance tolerance: configurable per item category (e.g. +/-2% for Class A, +/-5% for Class B, +/-10% for Class C).
Layout — Variance Investigation (bottom section, appears when variances exist)
Variance report: all items with counted qty differing from expected by more than tolerance. Per item: Item, Expected, Counted, Variance (#), Variance (%), Investigation Status (Pending / Investigating / Resolved). Investigation form per item: Probable Cause (dropdown: Receiving Error, Issue Error, Theft/Loss, Damage/Spoilage, Counting Error, System Error, Unknown), Resolution (Adjust Inventory to Counted, Recount, No Adjustment — system correct), Notes (free text), Investigated By, Resolved Date. Supervisor approval required for adjustments exceeding $ threshold.
Layout — Count Completion Actions
'Accept All Counts' (applies non-variance items automatically), 'Accept with Adjustments' (applies all counts including variance adjustments — requires supervisor approval), 'Recount' (generate new count sheet for items with variances for second count by different person), 'Cancel Count' (discard this counting session). On acceptance: GIP inventory quantities updated to counted values, IFCAP financial adjustment posted, audit trail created for all adjustments.
Data Source
ZVE SUPPLY COUNT wrapper -> GIP inventory (current quantities for expected values), GIP inventory adjustment (posting counted values), IFCAP financial adjustment (value of inventory changes), cycle count schedule configuration.
Unique Behaviors
Blind counting: during count entry, expected quantity is HIDDEN until the counter enters their counted value — this prevents the counter from just confirming the system value without actually counting. After entry, both values displayed with variance highlighted. Second-count protocol: for items with significant variance, best practice is to have a DIFFERENT person recount. System enforces: recount cannot be performed by same user who did initial count. Dollar-value impact: variance report shows not just quantity difference but dollar impact (variance qty x unit cost) to prioritize investigation of high-value discrepancies. Inventory valuation adjustment: large count adjustments may require financial reporting (write-down of inventory value) — flagged for fiscal service review. Cycle count compliance: system tracks whether all scheduled counts are being completed on time — compliance percentage reported on SU-09 reports. Wall-to-wall inventory: in addition to cycle counting, system supports annual wall-to-wall physical inventory (all items in a location counted at once) — generates comprehensive count sheets with all items at the selected location.

Trend analysis: all reports support 12-month rolling trend view showing improvement or degradation over time. Ad-hoc filtering: from any report table, click a cell to drill down (e.g. in Vendor Performance report, click vendor name to see all POs for that vendor).

Layout — Config Navigation (left sidebar 220px)
Tree-style navigation: Warehouse Locations, Distribution Points, Item Categories, Reorder Rules, Vendor Profiles, Fund Control Points, Cycle Count Schedules, Par Level Defaults, Equipment PM Templates, Alert Thresholds, User Permissions.
Layout — Config Form (center, 780px)
Per selected section, appropriate form fields with current values editable. Change highlighting (yellow background on modified values). 'Save' button shows diff preview before committing. 'Audit Note' prompt: every config change requires reason text. CAUTION banner at top: 'Changes to these settings affect supply chain operations for all users. Verify changes carefully before saving.'
Layout — Impact Preview (right sidebar 280px)
Shows what will be affected by current change: e.g. changing reorder point for an item shows: 'This will trigger immediate reorder for N items currently below new threshold' or 'This will affect par level calculations for N distribution points.' High-impact changes: require supervisor approval workflow (submitted for review before activation).
Key Configuration Sections
(1) Warehouse Locations: name, address, storage zones (ambient/refrigerated/frozen/secure/hazmat), bin numbering scheme. (2) Distribution Points: link to HOSPITAL LOCATION #44, par level capability Y/N, auto-replenishment Y/N. (3) Reorder Rules per item: Fixed Point (order when below threshold), Economic Order Quantity (calculated from demand, ordering cost, holding cost), Min-Max (reorder to max when hitting min), Just-In-Time (order only on demand). (4) Vendor Profiles: name, contact, terms, contract status, EDI capability, performance rating. (5) Fund Control Points: IFCAP control point names, officials, approval thresholds ($ amounts requiring different approval levels), annual budgets. (6) Cycle Count: ABC classification rules (define $ and movement thresholds for A/B/C), count frequency per class, variance tolerance per class. (7) Par Levels: default calculation method (average usage x lead time + safety stock), safety stock factor (1.0-2.0 multiplier). (8) Equipment PM: template definitions per equipment type (frequency, checklist items, estimated labor), compliance alert thresholds. (9) Alert Thresholds: days past expected delivery before PO alert, $ variance threshold for investigation, stock-out duration before escalation.
Data Source
ZVE SUPPLY CONFIG wrapper -> GIP configuration, IFCAP configuration, AEMS/MERS configuration files.
Unique Behaviors
CAUTION banner prominent. All changes audit-logged with reason, old value, new value, and user. Reorder rule changes: if changing from Fixed Point to EOQ for a category, system recalculates all reorder points for items in that category and previews the impact (how many items would trigger immediate reorder). Fund control point budget changes: require fiscal service approval (changes affect financial accountability). Vendor deactivation: deactivating a vendor warns if open POs exist for that vendor. PM template changes: affect future PM schedules but do not retroactively change completed PM records.
