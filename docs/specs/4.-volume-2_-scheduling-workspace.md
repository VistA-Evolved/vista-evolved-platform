# 1. Foundation
This document specifies every page, field, button, dropdown, and backend RPC touchpoint for the Scheduling Workspace. Scheduling is the recommended FIRST vertical slice for proving the modern client architecture because it has the richest existing RPC surface in VistA outside CPRS itself.
The January 2025 VSE Technical Manual documents over 150 RPCs in the SDEC and SDES/SDES2 namespaces. Two B-type context options secure these RPCs: SDECRPC (for SDEC namespace RPCs) and SDESRPC (for SDES namespace RPCs). Users must have access to these options to use the scheduling RPCs.
Core files: HOSPITAL LOCATION #44, SDEC APPOINTMENT #409.84, SDEC APPT REQUEST #409.85, CANCELLATION REASONS #409.2, SDEC RESOURCE #409.831, SDEC RESOURCE GROUP #409.832, SDEC CHECK-IN STEP STATUS #409.842, SDEC SETTINGS #409.98, RECALL REMINDERS #403.5.
Source: VSE Technical Manual for VS GUI Release 1.7.63.2, January 2025, Department of Veterans Affairs. All RPC names from Table 5 of that manual.

## 3.4 SCH-08: No-Show Flow
### Purpose
Mark a missed appointment. This is a high-caution action because no-show has operational and clinical significance (affects scheduling metrics, may trigger follow-up, and in some configurations may auto-rebook).
Source: No-Show RPC: SDES2 NO-SHOW. Undo: SDES2 UNDO NO-SHOW. The older PIMS scheduling rules (still operationally relevant) say: if patient has only one matching appointment not yet checked in, system can auto-no-show; if checked in or multiple appointments exist, user must confirm exact target.
