March 2026 | The Surgery workspace manages the complete perioperative lifecycle: case scheduling and OR board management, pre-operative assessment and clearance, intra-operative documentation (surgical record, anesthesia record, counts, implants, specimens), operative report dictation, post-operative care (recovery, orders, wound management), surgical quality and safety programs (NSQIP, SSI surveillance, implant tracking). VistA Surgery package (SR namespace, SURGERY #130) stores case data. Used by: surgeons, anesthesiologists, CRNAs, OR nurses (circulator and scrub), perioperative coordinators, surgical quality reviewers, and surgical coding specialists. References Vol 12 (Surgery) screen contracts.

# 1. Surgery Workspace Navigation Structure
The Surgery workspace uses a HYBRID navigation: the OR Schedule Board (SG-01) is the default landing page and serves as both a dashboard and navigation hub. Horizontal tabs below the board provide access to other surgery functions. Patient context is CASE-BASED: selecting a case on the OR Board loads that patient into the Patient Banner and opens the case-specific pages.

CASE-BASED CONTEXT: Unlike most workspaces where the user selects a PATIENT first, in Surgery the user typically selects a CASE from the OR Board (SG-01). Selecting a case loads both the patient into the Patient Banner AND the case-specific context (procedure, surgeon, OR room, scheduled time) into the case context bar that appears below the Patient Banner on all Surgery sub-pages. This dual context (patient + case) is necessary because a single patient may have multiple surgical cases (e.g. scheduled for knee replacement today and cataract surgery next month).

Specification
Page ID
SG-01
Template
Calendar (WF-01 5.7) + Dashboard (WF-01 5.4) HYBRID SPECIALIZED
Layout — Board Header (56px)
Left: 'OR Schedule' title + date selector (< Today > with calendar popup for jumping to future dates). Center: View mode toggles: Board (default room-time grid), List (flat table sorted by time), Timeline (Gantt-style per surgeon rather than per room). Right: Filter dropdowns: Service (All / General Surgery / Orthopedics / Neurosurgery / Cardiac / Vascular / Urology / Plastic / ENT / Ophthalmology / Podiatry / Oral Surgery / Gynecology), Status (All / Scheduled / In-OR / Recovery / Complete / Cancelled / Delayed), Surgeon (provider search). Layout — KPI Strip (40px, below header)
Horizontal metrics bar: Cases Today: N (total scheduled), In Progress: N (currently in OR), Completed: N, On-Time Start Rate: N% (first cases starting within 10 minutes of scheduled time — green >90%, amber 75-90%, red <75%), Average Turnover: N min (time between case-out and next case-in — green <30 min, amber 30-45, red >45), Pending Recoveries: N (patients in PACU). Each metric clickable to filter board.
Layout — Room-Time Grid (main area, fluid height)
Rows: OR rooms (OR-1, OR-2, ... OR-N, configurable per facility — typically 4-20 ORs). Each row = one physical operating room. Columns: time blocks spanning operating hours (default 06:00 - 18:00, configurable per facility, 30-minute grid increments with 15-minute visual subdivisions). Current time: vertical red line that advances in real-time. Cases rendered as BLOCKS within their row: Block width = estimated case duration (proportional to time scale). Block content: Patient last name (bold, truncated), Procedure (abbreviated, 2 lines max), Surgeon last name, Block color by STATUS: Scheduled (solid blue), Holding/Pre-Op (light blue with striped pattern), In-OR (green with subtle pulse animation indicating active surgery), Recovery/PACU (yellow), Complete (gray), Cancelled (light gray with strikethrough text), Delayed (red border + amber fill with delay duration badge, e.g. '+45min'). Block height: fills the row height minus 4px padding. Overbooked time (two cases assigned to same OR at overlapping times): split block with conflict warning icon.
Layout — Empty Slot Behavior
Clicking an empty time slot in any OR row opens a quick-schedule popover: Patient search field + Procedure field + Surgeon field + Duration estimate + 'Book' button. This is the fast-path for adding cases. Full booking (SG-02) available via 'Add Case' button for complex cases.
Layout — Case Block Interactions
Click case block: opens Case Detail flyout (right 350px panel) showing full case information (patient, procedure, team, times, status, pre-op checklist status, special requirements). Double-click: navigates to full case detail page (SG-02b). Drag case block to different time or different OR room: initiates RESCHEDULE with confirmation dialog ('Move [Patient] [Procedure] from OR-3 08:00 to OR-5 10:00?'). Right-click case block: context menu with actions: View Detail, Edit, Check In (Pre-Op), Start Case, End Case, Send to PACU, Complete, Cancel (with reason), Print Face Sheet.
Layout — Bottom Panel (collapsible, 200px)
Two tabs: (A) 'Holding Area' — patients checked in for surgery but not yet in OR: Patient, Procedure, Surgeon, Arrival Time, Pre-Op Status (checklist completion percentage bar), Assigned OR, Scheduled Time, Minutes Until Scheduled. Sorted by scheduled time. Drag from Holding to OR grid to advance status. (B) 'PACU' — patients currently in recovery: Patient, Procedure, In PACU Since, Aldrete Score (auto-updated from SG-08 PACU page), Discharge Criteria Met (Y/N), Disposition (Ward/Home/ICU). Sorted by PACU time (longest first).
Data Source
SURGERY #130 (case data: patient, procedure, dates, times, status, team), OPERATING ROOM #131.7 (OR room definitions), block time assignments, via ZVE SURG BOARD wrapper. Real-time status updates via WebSocket or 30-second polling.
Unique Behaviors
REAL-TIME BOARD: status changes (case start, case end, PACU arrival) update the board within seconds for all users viewing it. The OR Board is typically displayed on large wall-mounted monitors in the surgical suite for the entire perioperative team to reference. Delay cascade visualization: when a case runs longer than estimated, subsequent cases in the same OR automatically shift right on the timeline and display delay badges. The delay propagates visually so the team can see which downstream cases are affected and by how much. Block time overlay: toggle to show surgeon block time allocations as semi-transparent colored backgrounds on the grid (e.g. Dr. Smith has OR-3 blocked Mondays 07:00-15:00, shown as light blue background). Cases scheduled outside a surgeon's block are visually distinct. Add-on management: unscheduled urgent/emergent cases are added via the 'Add Case' workflow; the board highlights available OR time slots that can accommodate the estimated duration. Turnover timer: when a case ends, a turnover clock starts in the now-empty OR slot, counting up until the next case's patient enters the OR. This drives turnover time tracking. Print: generates formatted daily OR schedule for posting (includes patient names, procedures, surgeons, times, special requirements per case).

Data Source

Unique Behaviors
LATERALITY is the SINGLE MOST IMPORTANT patient safety element in surgical booking. Wrong-site surgery is a 'Never Event' (should never occur). The system enforces: (1) Laterality required for all extremity, eye, ear, kidney, lung, and other paired-organ procedures, (2) Explicit confirmation dialog, (3) Laterality displayed prominently on all downstream documents (consent, pre-op checklist, OR Board block, specimen label). Equipment availability check: real-time query of OR equipment inventory — if requested C-Arm is already assigned to another case at the requested time, system warns and suggests alternative times or rooms. Implant pre-ordering: for orthopedic cases requiring specific implant sizes (e.g. total knee components), booking triggers vendor notification to ensure implants are available on the day of surgery. Surgical History Display: during booking, system shows patient's prior surgeries at this facility (from SURGERY #130) so surgeon can reference prior operative reports and complications.

## SG-03: Pre-Operative Assessment
Aspect
Specification
Page ID
SG-03
Template
Form (WF-01 5.3) COMPLEX
Layout — H&P Summary Panel (left 45%)
Auto-pulled patient clinical summary: Active Problems (#9000011 — relevant surgical comorbidities highlighted: diabetes, cardiac disease, COPD, bleeding disorders, renal insufficiency), Current Medications (from Pharmacy #52/#55 — anticoagulants/antiplatelets flagged RED with 'Hold Before Surgery?' prompt: Warfarin hold 5 days, Plavix hold 7 days, Aspirin per surgeon preference, DOACs per agent-specific timeline), Allergies (#120.8 — latex allergy triggers latex-free OR setup flag), Recent Relevant Labs (auto-pulled from Lab #60 based on procedure type: CBC within 30 days, BMP/CMP within 30 days, PT/INR within 7 days for anticoagulated patients, Type & Screen within 3 days if blood products anticipated, HbA1c for diabetics, Pregnancy test for females of childbearing age within 7 days). Lab Freshness Indicators: green (current), amber (approaching expiration), red (expired — needs repeat).
Layout — Pre-Op Checklist (right 55%)
Structured checklist per Joint Commission Universal Protocol + VHA surgical safety standards. Each item: checkbox + status + responsible person + timestamp when completed. Items: (1) Informed Consent: signed Y/N — link to consent document (SG-13). 'Not Signed' = RED block preventing case from proceeding. (2) Surgical Site Marked: Y/N + by whom. Required for all lateralized procedures. Mark must match consent and booking laterality. (3) NPO Status: 'Nothing by mouth since: [time]' — auto-calculated hours. Minimum 2h clear liquids, 6h light meal, 8h full meal per ASA guidelines. Insufficient NPO: RED warning 'Patient not NPO long enough — risk of aspiration. Consult anesthesia.' (4) Pre-Op Labs Reviewed: Y/N + reviewer name. System checks freshness per above. (5) Blood Products Ordered: N/A or 'Type & Screen sent / N units crossmatched' per BB-01. (6) Imaging Reviewed: Y/N for procedures requiring pre-op imaging (e.g. orthopedic — X-rays, CT for navigation). (7) Anesthesia Clearance: ASA Classification (I-VI) assigned by anesthesiologist, airway assessment (Mallampati I-IV), pre-anesthesia evaluation complete Y/N. (8) Antibiotic Prophylaxis: appropriate antibiotic ordered per SCIP (Surgical Care Improvement Project) guidelines — drug, dose, timing (within 60 minutes of incision, 120 for vancomycin/fluoroquinolone). Auto-suggests per procedure type. (9) VTE Prophylaxis: DVT prevention ordered per VHA clinical guideline — mechanical (SCDs) and/or pharmacologic (heparin/enoxaparin). Auto-suggests per risk level. (10) Pregnancy Test: for females 12-55 — result documented. Positive = CASE CANCELLED pending OB evaluation.
Layout — Risk Score Panel (bottom)
Surgical Risk Score calculation: input variables auto-populated from patient data (age, sex, ASA class, functional status from nursing assessment NUR-02, emergency status, wound class, procedure CPT). Output: predicted mortality (%), morbidity (%), specific complication probabilities (SSI, pneumonia, cardiac event, DVT/PE, renal failure, UTI, return to OR, readmission). ACS NSQIP Risk Calculator integration. Visual: patient's risk displayed as percentile bar relative to national average for this procedure — 'Your patient is at the Nth percentile for predicted mortality for this procedure.' Red zone (high risk) prompts: 'Consider preoperative optimization: [suggestions based on modifiable risk factors].'
Layout — Clearance Summary & Actions
Bottom action bar: 'Pre-Op Complete' button (all checklist items must be green/complete — button DISABLED if any required item incomplete with tooltip showing incomplete items). 'Save Progress' (for partial completion across multiple visits). 'Print Pre-Op Face Sheet' (formatted summary for OR day: patient info, procedure, team, allergies, labs, checklist status, special requirements).
Data Source
ZVE SURG PREOP wrapper -> SURGERY #130 pre-op fields. Cross-workspace data pulls: Problems #9000011, Meds #52/#55, Allergies #120.8, Labs #60, Blood Bank via BB-01, Nursing Assessment via NUR-02, Consent via SG-13. Risk calculator: ACS NSQIP model coefficients.
Unique Behaviors
SAFETY-CRITICAL PAGE: every item on the pre-op checklist exists because its absence has caused patient harm in documented surgical safety incidents. The checklist is the last systematic safety barrier before the patient enters the OR. The 'Pre-Op Complete' button being DISABLED until all items are green is a HARD STOP — it cannot be overridden by any user role except in documented life-threatening emergency (which creates a special exception record with attending surgeon attestation). Anticoagulation management: system maintains a medication-specific hold-before-surgery table (Warfarin 5d, Clopidogrel 7d, Rivaroxaban 48h, etc.) and cross-references against patient's active medications to flag each anticoagulant with recommended hold timeline. SCIP antibiotic timing: VHA-tracked quality metric. System alerts circulating nurse when it is time to administer pre-op antibiotic (60 minutes before planned incision, based on scheduled case start time minus prep time). Late antibiotic = quality variance.

Personnel changes timestamped.
Layout — Counts Section (CRITICAL SAFETY)
Three count categories displayed prominently: SPONGES (Initial Count + Additions + Final Count = Expected Total, Actual Count, Match: YES green / NO red), INSTRUMENTS (same structure), NEEDLES / SHARPS (same structure). Count timing: (1) Before incision (initial), (2) Before closure of any body cavity, (3) At skin closure (final), (4) At any personnel relief. COUNT STATUS BANNER: GREEN 'All Counts Correct' or RED 'COUNT DISCREPANCY — DO NOT CLOSE' with flashing alert. Incorrect count triggers: mandatory X-ray before closure, surgical team search, incident report. System BLOCKS 'Procedure End' timestamp until counts are reconciled or exception documented (with attending surgeon attestation that X-ray performed and cavity searched).
Layout — Implants Section
Table: Device Description, Manufacturer, Catalog Number, Lot Number, Serial Number (if applicable), UDI (Unique Device Identifier — scanned from device packaging), Size/Specification, Quantity, Anatomic Location Implanted. Each implant barcode-scanned from packaging label capturing UDI data automatically. 'Add Implant' button for each device used. Photo of implant label: device camera capture for documentation. Total implant count displayed.
Layout — Specimens Section
Table: Specimen Label (Part A, B, C...), Anatomic Description (e.g. 'Left breast, lumpectomy specimen — superior margin inked blue, anterior margin inked green'), Type (Tissue for Permanent / Frozen Section / Culture / Cytology), Sent to (AP / Microbiology), Time Sent, AP Accession # (auto-linked when AP receives and accessions in AP-01). Frozen section: timestamp when sent + timestamp when pathologist calls result + frozen diagnosis text — tracked for frozen section turnaround time quality metric.
Layout — Events / Complications Section
Chronological event log during case: Timestamp + Event Description. Pre-populated events: 'Antibiotic administered' (from pharmacy/anesthesia record), 'Tourniquet up/down' (with duration), 'Blood products administered' (from BCMA/BB transfusion record). Manual events: 'Specimen obtained', 'Implant placed', 'Complication occurred'. Complications dropdown: None, Hemorrhage (>EBL threshold), Organ Injury (specify), Cardiac Event, Airway Difficulty, Equipment Malfunction, Transfusion Reaction, Anaphylaxis, Other (free text). EBL (Estimated Blood Loss): numeric entry in mL.
Data Source
ZVE SURG INTRAOP wrapper -> SURGERY #130 (operative case fields: times, personnel, counts, implants, specimens, complications, EBL). Implant UDI: scanned from device packaging linked to FDA GUDID database. Specimen AP link: triggers AP-01 accession via ZVE AP ACCESSION wrapper. Pharmacy/BCMA: antibiotic timing from medication administration records.
Unique Behaviors
COUNT SYSTEM is the most critical safety feature on this page. Retained surgical items (sponges, instruments, needles left inside patient) are a 'Never Event' per CMS and a devastating complication. The system provides an ABSOLUTE BLOCK on case closure when counts are discrepant — this is intentionally inflexible. The only override is the documented exception pathway (attending surgeon attests: X-ray performed, cavity searched, decision made to proceed despite discrepancy with documented rationale). This override creates an incident report automatically. UDI scanning: FDA mandates UDI tracking for all implantable devices. The system captures UDI by barcode scan, which auto-populates manufacturer, catalog#, lot#, serial# from the FDA GUDID database — eliminating manual transcription errors. UDI data flows to both the Implant Registry (SG-12) and Supply/Prosthetics for inventory management. Specimen-AP link: when specimen is logged in SG-04, an electronic notification is sent to AP (Vol 6 Supplement) with patient identity and clinical information — when AP accessions the specimen (AP-01), the accession number flows back to link the surgical case to the pathology report. This bidirectional link ensures: surgeon can see pathology results from their case, and pathologist can see the surgical context when interpreting the specimen.

Unique Behaviors

Auto-population: the operative report auto-populates approximately 40-60% of content from booking and intra-operative record data, significantly reducing dictation time. Surgeon reviews, modifies as needed, and adds the narrative description of the actual procedure. Report linkage: the signed operative report appears in: (1) Clinical workspace CL-10 Progress Notes for the patient's chart, (2) Billing workspace BL-02/BL-03 for surgical coding, (3) Radiology workspace if imaging was performed during the procedure. Addendum: after sign-out, if corrections or additions are needed (e.g. pathology final report differs from frozen section preliminary), surgeon creates an Addendum — original report preserved unchanged, addendum linked below per Audit/Irreversibility Matrix C/L3.

(9) Laboratory: morning labs (CBC, BMP), other labs per procedure (e.g. hemoglobin Q6h post-op for bleeding risk procedures, troponin for cardiac surgery). (11) Consults: physical therapy, occupational therapy, nutrition, social work/discharge planning (auto-triggered for complex patients per SW-02 rules). (12) Discharge Criteria: when applicable for same-day surgery — checklist of criteria that must be met before discharge (Aldrete >=9, vital signs stable, pain controlled, no active bleeding, ambulatory, tolerating fluids, voided, responsible adult present).
Layout — Review & Sign (bottom)
Order summary: compiled list of all orders from all sections. Each order: text, dose/frequency if medication, responsible service. 'Review All' expands full order text for each. 'Sign Orders' button: single e-signature covers all orders in the set. On sign: orders are created in ORDER #100, flowing to: Pharmacy (medications), Lab (lab orders), Nursing (nursing orders, vitals, I&O), Nutrition (diet), and relevant services (PT, OT, SW). Simultaneous multi-workspace order creation is the key efficiency — surgeon signs once, system routes to all destinations.
Data Source
ZVE SURG POSTOP wrapper -> ORDER #100 order set creation. Order sets: pre-defined in order set configuration (SG-15 Config). Individual orders flow to: PHARMACY PATIENT #55 (meds), LABORATORY TEST #60 (labs), DIETS #115 (diet), nursing task files (nursing orders), REQUEST/CONSULTATION #123 (consults).
Unique Behaviors
ORDER SET EFFICIENCY: without order sets, a surgeon would need to enter 15-30 individual orders after every case. Order sets pre-configure the standard post-op regimen, and the surgeon modifies as needed. This reduces post-op order time from 15-20 minutes to 2-3 minutes while ensuring nothing is forgotten. SCIP antibiotic auto-discontinue: system calculates the auto-discontinue time (24 hours post-op for most procedures) and includes this in the antibiotic order — preventing extended unnecessary antibiotic courses (an antibiotic stewardship quality metric). PCA safety: PCA order includes safety parameters (4-hour limit, lockout interval) that are programmed into the smart pump (BD Alaris integration per Device Catalog) — BCMA verifies PCA settings match the order. VTE prophylaxis: VHA has a national clinical guideline for post-surgical VTE prevention — the order set auto-suggests the appropriate agent and timing based on procedure type and patient risk factors (auto-pulled from SG-03 risk assessment).

SG-10
Surgical Quality Dashboard
Dashboard (WF-01 5.4)
NSQIP quality dashboard: Observed vs Expected ratios for key outcomes: Mortality O:E, Morbidity O:E, SSI O:E, Pneumonia O:E, Readmission O:E. O:E < 1.0 = better than predicted (green), 1.0 = as expected (amber), >1.0 = worse than predicted (red). Trend charts: 12-month rolling O:E ratios with statistical control limits. NSQIP (National Surgical Quality Improvement Program): gold standard for risk-adjusted surgical quality measurement. VA has participated since NSQIP's inception (originally VA-NSQIP before ACS adopted it nationally). O:E ratio is the KEY metric: adjusts for patient risk so facilities performing higher-risk cases are not unfairly penalized. Outlier detection: O:E significantly >1.0 triggers quality review — what is causing worse-than-expected outcomes? Peer comparison: facilitates benchmarking against other VA facilities and private sector hospitals.
SG-11
Surgical Site Infection Tracking
Worklist + Form
SSI surveillance per CDC NHSN criteria: Case list with SSI monitoring window (30 days post-op for most procedures, 90 days for procedures with implants). Columns: Patient, Procedure, Procedure Date, Monitoring End Date, SSI Identified (Y/N), SSI Type (Superficial Incisional / Deep Incisional / Organ/Space), Pathogen(s), Date Identified. SSI documentation form: NHSN event criteria (purulent drainage, positive culture, wound opened by surgeon, signs/symptoms + positive culture), classification, treatment. SSI rates: per procedure category, compared to NHSN national benchmarks, displayed as Standardized Infection Ratio (SIR).
ZVE SURG SSI wrapper -> Surgery infection tracking data + Lab culture results.
CDC NHSN: national infection surveillance network. SSI is the most common healthcare-associated infection after surgery. Monitoring window: system auto-generates follow-up list of patients within their surveillance window, flagging those with positive wound cultures or surgical wound complications. Pathogen tracking: identifies organisms causing SSI (MRSA vs MSSA vs gram-negative, etc.) for antibiotic stewardship. SIR: risk-adjusted rate (similar to O:E ratio) accounting for procedure type, wound class, ASA class, duration — SIR <1.0 is better than national average.
