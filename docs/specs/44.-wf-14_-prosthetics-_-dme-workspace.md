WF-14: Prosthetics / DME Workspace
Document 14 of 16 | ~13 Pages | Prosthetics Dashboard, Patient Prosthetic Record, Purchase Orders, VA Form 2529-3, HISA Applications, Home Oxygen, Inventory, NPPD Reporting, Repairs, Recalls, Vendors, Reports, Configuration
March 2026 | The Prosthetics/DME workspace manages the complete lifecycle of prosthetic devices, durable medical equipment (DME), and sensory aids for veterans. VistA Prosthetics package (RMPR/RMPO namespaces, V3.0, TM revised November 2025, Basic UM revised October 2024) uses files #660-669.9. Key files: PROSTHETICS #669 (patient prosthetic record with VA Form 10-2319 data), PURCHASE ORDER #668 (prosthetic purchasing), #664 (IFCAP 1358 purchasing bridge). Every prosthetic item must have a PSAS HCPCS (Healthcare Common Procedure Coding System) code for: (1) item identification, (2) NPPD (National Prosthetic Patient Database) reporting to Congress, (3) cost tracking, (4) utilization analysis. NPPD per Prosthetics TM: 'HCPCS are mapped to NPPD Groups/Lines, providing a means to analyze, validate, summarize, and report usage and cost at medical center and national levels. Data is rolled up at each facility and transmitted to Hines.' NPPD is a CONGRESSIONAL REPORTING REQUIREMENT. IFCAP integration for all purchasing: IFCAP manages funds distribution, control point activity, accounting, and procurement. Home Oxygen managed under RMPO namespace with separate administrative module. Prosthetic Inventory Package (PIP) with barcode tracking using Zebra label printers. Security keys: RMPRMANAGER, RMPRSUPERVISOR. This workspace is used by: Prosthetic Representatives (PSAS staff), Prosthetic Purchasing Agents, Prosthetic Technicians, Home Oxygen Coordinators, and Prosthetic Service Chiefs. References Vol 16 (Prosthetics/DME) screen contracts.

Unique Behaviors
CONSULT-DRIVEN WORKFLOW: prosthetic items are not self-ordered by Prosthetics staff; they are REQUESTED by clinical providers via consult (Request/Consultation #123) from the Clinical workspace. The consult flows to the Prosthetics service, where the Prosthetic Representative evaluates the request, verifies eligibility, and initiates the purchasing/issue process. The Pending Consults panel is the primary intake queue. BUDGET PROMINENCE: Prosthetic budget management is critical — prosthetics is one of the largest expenditure categories in many VA facilities ($800M+ nationally per year). The budget card is prominently displayed because overspending requires VISN-level authorization and underspending may indicate access problems (veterans not receiving needed items). REPLACEMENT SCHEDULING is a PROACTIVE feature: rather than waiting for equipment to fail, the system tracks expected service life (e.g. below-knee prosthesis 3-5 years, wheelchair 5-7 years, hearing aid batteries monthly) and alerts when replacement is approaching, enabling PROACTIVE outreach to the veteran. Per Vol 16: 'system calculates expected replacement dates based on item lifespan, upcoming replacements trigger proactive outreach.'

Chronological table of ALL prosthetic transactions for this patient (equivalent to VA Form 10-2319 display — per Prosthetics Basic UM: 'Display/Print Patient 2319' option). Transaction Types: New Issue (item first provided), Repair (item repaired — in-house or vendor), Replacement (end-of-life item replaced with new), Rental (temporary item while permanent is being fabricated/ordered), Return (item returned to VA — no longer needed, upgrading), Loaner (temporary item during repair). Per transaction: Date, Type, Item (HCPCS + description), Cost, Vendor, Purchasing Agent, Status (Open/Complete), NPPD Line/Group. Sortable by date (default newest first) or by item type. Export: full 2319 record printable as formatted VA Form 10-2319.
Layout — Limb Loss / Amputation History (expandable section)
For amputee veterans: Amputation Level (e.g. Right Below-Knee, Left Above-Elbow), Laterality, Date of Amputation, Etiology (combat injury, disease, trauma), Prosthetic Prescription History (all limbs prescribed: type, date, fitting notes, functional outcome). Limb clinic visit history. Gait analysis references. K-Level (functional level K0-K4 determining prosthetic technology authorization: K0=non-ambulatory, K1=household ambulator, K2=limited community, K3=community ambulator, K4=exceeds basic ambulation — per Medicare/VA guidelines). K-Level determines which prosthetic components are authorized (e.g. microprocessor knees require K3+).
Layout — Letters / Correspondence (expandable section)
Letters on file for this patient: appointment letters, quotation requests (FL ADP 10-90 per Prosthetics Basic UM: 'An estimate on the above-listed items is requested. YOUR QUOTATION DOES NOT CONSTITUTE A PURCHASE ORDER'), approval letters, denial letters with appeal rights, recall notification letters. Per Prosthetics Basic UM: 'Appointments, and letters on file. You can select a Letter on File to be viewed.'
Layout — Actions (bottom bar)
'Enter New Request' (create PO for this patient — opens PR-03 with patient pre-loaded), 'Process 2529-3' (generate authorization form — opens PR-04), 'Schedule Appointment' (link to Scheduling SC-01 for prosthetic clinic visit), 'Request Repair' (opens PR-09 Repair entry for selected equipment item), 'HISA Application' (opens PR-05 for home modification), 'Home Oxygen' (opens PR-06 for O2 management), 'Print 2319' (formatted print of complete prosthetic patient record).
Data Source
PROSTHETICS #669 (patient prosthetic record — the 2319 data per TM: 'Prosthetics package files are in the range of 660 to 669.9'), PURCHASE ORDER #668 (linked POs), IFCAP #442 (financial data), PSAS HCPCS code tables (item identification), PATIENT #2 (demographics), eligibility data from Registration PT-04, K-Level assessment data — via ZVE PROS PATIENT wrapper.
Unique Behaviors
LONGITUDINAL EQUIPMENT RECORD: this page shows the ENTIRE history of prosthetic devices and services provided to the veteran over their lifetime of VA care. For veterans who have been receiving prosthetics for decades (e.g. Vietnam-era amputees receiving replacement limbs every 3-5 years for 50+ years), this record may contain dozens of transactions. The record is NEVER purged for active patients. Per Prosthetics TM: '#664 records purged with same criteria as IFCAP #442 annual purging (8 years)' for financial records, but patient clinical prosthetic history (#669) is retained. K-LEVEL SIGNIFICANCE: the K-Level functional classification directly determines which prosthetic technologies are authorized. A K3 patient (community ambulator) is eligible for advanced components (microprocessor knees like C-Leg/Genium at $50,000-100,000+) that a K1 patient (household only) is not. The K-Level assessment is performed by the prosthetic prescriber (physiatrist, orthopedic surgeon, or prosthetist) and documented here. Disputes about K-Level classification are among the most consequential prosthetic clinical decisions. HCPCS MANDATORY: per NPPD requirements, EVERY prosthetic item must have a PSAS HCPCS code. Items without HCPCS codes cannot be properly reported to NPPD and represent a data quality issue (flagged in AN-07 Data Quality Monitor). For complex prosthetic limbs, multiple HCPCS codes may apply to a single limb (base code + component codes for knee, foot, socket, suspension, etc. — per NPPD FAQ: 'The first PSAS HCPCS should be the basic L code used for the limb, the second master item HCPCS will be the next code on the limb prescription. This procedure should be used for each subsequent item.').

Data Source

Unique Behaviors
IFCAP INTEGRATION is the financial backbone: every prosthetic purchase creates an IFCAP obligation against a fund control point. The obligation reduces available budget balance. When the item is received and payment processed, the obligation converts to an expenditure. This double-entry financial tracking ensures accurate budget management. Per Prosthetics TM: 'IFCAP manages funds distribution, control point activity, accounting, and procurement.' MULTI-HCPCS PO for prosthetic limbs: a single below-knee prosthetic may have 5-10 HCPCS codes: L5301 (below knee, molded socket, SACH foot), L5649 (suction socket insert), L5679 (addition, below knee, external keel SACH foot), L5685 (addition, multi-axial ankle), L5845 (addition, stance flexion feature — mechanical), L5856 (addition, endoskeletal knee-shin system — pneumatic), etc. Each HCPCS line has its own cost, vendor, and NPPD record. The PO aggregates all lines for a single procurement action. WAIVER AND EXCLUDED TRACKING: per Prosthetics GUI UM, items that are prescribed outside of mandatory BPA/contracts require either a Waiver (item is on a national contract but this specific patient needs a non-contract item for medical reasons) or an Excluded Notice (item is not covered by any national contract). These designations affect NPPD reporting and procurement compliance monitoring.

PR-04
VA Form 2529-3 Processing
Form (WF-01 5.3)
Official prosthetic authorization form. Auto-populated: Patient demographics (from #2), Item(s) prescribed with HCPCS (from consult or manual entry), Prescribing Physician (from consult ordering provider or manual), Clinical Justification (from consult clinical information), Delivery method (in-house fabrication / commercial vendor / VA contract per Prosthetics TM: 'Process Form 2529-3 (PS menu)'). Form generation: electronic VA Form 2529-3 with e-signature capability. Approval workflow: Prosthetic Representative reviews -> Supervisor approval (if required by cost/type) -> Form signed -> links to PO creation (PR-03) for purchasing. Print: formatted 2529-3 for paper signature if electronic not available. Status tracking: Pending/Approved/Denied with reason.
ZVE PROS 2529 wrapper -> prosthetic authorization data linked to #669 + #668. Per Prosthetics Basic UM: 'PS Process Form 2529-3' menu.
VA Form 2529-3 is the LEGAL AUTHORIZATION for prosthetic provision. Without a signed 2529-3 (or electronic equivalent), VA cannot purchase or issue a prosthetic item. The form documents: medical necessity, prescribing physician approval, item specification, and authorization chain. For complex/high-cost items (myoelectric upper extremity, C-Leg/Genium, power wheelchair), the 2529-3 requires additional clinical documentation (functional assessment, K-Level, specific component justification, letters of medical necessity from prescriber). The electronic form generation eliminates the historically paper-heavy process and enables tracking of authorization-to-delivery timelines.
PR-05
HISA Application
Form (WF-01 5.3)
Home Improvement and Structural Alterations grant application. Fields: Patient, Disability (SC or NSC), Proposed Modifications (ramp, widened doorways, roll-in shower, stair lift, bathroom grab bars, kitchen modifications, etc. — from standardized modification catalog), Cost Estimate (contractor quote attached), Contractor Information, Previous HISA Grants (system displays lifetime grants awarded with cumulative total: statutory limits $6,800 for SC / $2,000 for NSC — remaining balance prominently displayed in GREEN if available, RED if exhausted), Home Assessment Report (optional: occupational therapy evaluation of home environment documenting barriers and recommended modifications). Approval workflow: HISA Coordinator review -> Medical Director review -> approve/deny with authorization letter. Denial: generates letter with appeal rights. Physical inspection: post-modification inspection checklist to verify work completed per specification.
ZVE PROS HISA wrapper -> HISA application data within prosthetics files.
HISA LIFETIME LIMITS are statutory (set by Congress, not VA policy): $6,800 for SC veterans, $2,000 for NSC veterans. These limits have not been updated in decades and are increasingly insufficient for modern home modifications (a basic wheelchair ramp can cost $3,000-5,000). The system tracks cumulative grants against these limits with ZERO tolerance for exceeding them. If a veteran has received $5,000 in prior HISA grants (SC), only $1,800 remains — the system displays this prominently and prevents processing a grant application exceeding the remaining balance. Cost estimate verification: the system can compare the contractor's estimate against national average costs for similar modifications (from historical HISA data) to identify potentially inflated estimates.
PR-06
Home Oxygen Management
Detail View + Form SPECIALIZED
RMPO namespace — separate administrative module for home oxygen. Patient O2 record: Current Prescription (flow rate L/min, duration: continuous/exertion/sleep, date prescribed, prescribing physician), Equipment on Hand (concentrator model/serial, portable units, cylinder count/size, delivery system: nasal cannula/mask/transtracheal), Vendor (DME company providing equipment and oxygen), Delivery Schedule (bulk liquid delivery or cylinder exchange frequency), Qualifying Criteria (SpO2 result from qualifying test — per Medicare/VA: SpO2 <=88% on room air qualifies, or <=89% with cor pulmonale/polycythemia/CHF per Prosthetics ICD-10 Release Notes: 'Add/Edit Home Oxygen Patient' option with ICD-10 diagnostic codes), Prescription Renewal (annual re-evaluation required — system calculates renewal date and generates tickler), Cost Tracking (monthly cost, annualized cost, comparison to alternative modalities). Per Prosthetics TM: 'Home Oxygen Main Menu (HO) under RMPO namespace' is a separate submenu within Prosthetics.
ZVE PROS OXYGEN wrapper -> RMPO home oxygen data within prosthetics files. SpO2 qualifying data from Lab #60 or Vitals #120.5.
Home oxygen is among the HIGHEST-VOLUME prosthetic expenditures nationally. Many thousands of veterans receive home O2. Prescription renewal is critical: Medicare/VA rules require periodic re-evaluation to confirm continued medical necessity. The system generates renewal reminders to the prescribing physician and the prosthetic oxygen coordinator. If renewal not completed by expiration date: system flags for potential service interruption (patient safety — oxygen-dependent veterans CANNOT have service interrupted without medical reassessment). Vendor management: tracks deliveries, equipment swaps (concentrator maintenance, cylinder exchanges), and emergency service calls. Cost optimization: system can identify patients on expensive modalities (liquid O2) who may be candidates for less expensive alternatives (concentrator with portable cylinders) based on usage patterns and clinical need.
PR-07
Prosthetic Inventory (PIP)
Worklist + Form SPECIALIZED
Prosthetic Inventory Package. Stock management with barcode tracking. Inventory table: Item (HCPCS + description), Location (storage location code), Quantity on Hand, Cost per Unit, Total Value, Barcode Label Status (labeled Y/N), Last Activity Date. Per PIP documentation: 'All items will need a HCPCS code and Location before creating the barcode label.' Issue from Stock: scan barcode label (Zebra printer-generated per PIP: 'recommended Zebra printer') to issue item to patient — updates inventory and creates patient 2319 record simultaneously. Receive to Stock: scan received items into inventory from PR-03 receiving. Physical Count: worksheet generation (per PIP: 'Print Stock Worksheet — conduct a Physical Count at least monthly during testing or if discrepancies arise, conduct one on the most active HCPCS Codes'), count entry with variance investigation. Stock reports: balance by HCPCS (PI option), balance by location (PL option), transaction history (PS option), items not issued within 30 days (P3 option).
ZVE PROS INVENTORY wrapper -> PIP inventory files (converted per Patch RMPR*3*61). Barcode scanning: Zebra ZD620 or compatible label printer (per Device Catalog).
BARCODE-DRIVEN OPERATIONS: per PIP documentation, 'once patch 61 is installed, the Issue from Stock option will only work with a barcode label. A barcode must be scanned or you can manually enter a barcode sequence number.' VistA Evolved maintains this barcode-centric workflow using the Zebra label printers specified in the Device Catalog. Each inventory item has a unique barcode label printed from the Print All Barcode in a Location (AL) option. Scanning at issue time ensures: correct item identification (right HCPCS), accurate inventory decrement, and automatic patient record update. MONTHLY PHYSICAL COUNT: PIP mandates at least monthly physical counts for active HCPCS codes. Discrepancies between system balance and physical count require investigation (possible causes: unrecorded issue, receiving error, theft/loss, system error). Count results are documented for audit trail.
PR-08
NPPD Reporting
Report + Form SPECIALIZED
National Prosthetic Patient Database reporting. NPPD Worksheets: per NPPD UM, system generates worksheets showing all prosthetic transactions for the reporting period with: HCPCS, NPPD Group/Line mapping, patient data, cost, vendor, purchasing agent. Data Validation: before quarterly submission, validation checks run: missing HCPCS codes (no HCPCS = not reportable), missing costs (zero-cost records flagged), HCPCS-to-Group/Line mapping errors (HCPCS not mapped to an NPPD category), duplicate records, patient eligibility verification. Validation errors must be corrected before submission. NPPD Extract: generates data file for transmission to Hines Information Technology Center. Per Prosthetics TM: 'Data is rolled up at each facility and transmitted to Hines.' Quick Edit 2319 Record (QED2): rapid correction of NPPD data elements on patient records. Print NPPD Single Line Detail (SL): detailed report per NPPD Group/Line showing all transactions. Per NPPD: 'data is returned to the VISNs on Compact Disks and formatted for Microsoft Access database program. Through this program, pre-defined and ad hoc queries can be made.' VistA Evolved replaces the Access database approach with integrated analytics (AN-03 Report Builder can query NPPD data directly).
ZVE PROS NPPD wrapper -> NPPD data within prosthetics files, PSAS HCPCS mapping tables. Per NPPD UM: 'ND NPPD Tools' menu options.
CONGRESSIONAL REPORTING REQUIREMENT: NPPD data is used to justify the VA Prosthetics budget to Congress. Data accuracy directly impacts budget allocation. Per NPPD catalog.data.gov entry: 'Its objective was to enable clinical reviews to increase quality, reduce costs, and improve efficiency of the Prosthetics program.' QUARTERLY CYCLE: data is reported quarterly. Facilities have a submission window (typically first 2 weeks of the quarter for prior quarter data). Missing the submission window: generates national-level visibility (VISN directors notified). The NPPD tab provides real-time visibility into submission readiness throughout the quarter, not just at deadline time. HCPCS SYNONYM MANAGEMENT: per NPPD UM, 'HCPCS Synonyms (AE) from the NPPD Tools menu' allows mapping local item names to standard HCPCS codes. This is critical for facilities that use non-standard item descriptions in their local PROS ITEM MASTER.
PR-09
Repair Tracking
Worklist + Form
Repair requests for existing prosthetic equipment. Worklist: Patient, Item (currently issued equipment from PR-02 active equipment), Problem Description, Repair Status (Requested/Approved/Sent to Vendor/In Repair/Returned/Complete), Date Requested, Date Sent, Expected Return, Loaner Issued (Y/N — if the veteran's primary device is out for repair, was a loaner provided?). Repair entry form: Item (select from patient's active equipment), Problem Description, Repair vs Replace recommendation (minor repair / major repair / beyond repair -> replace), Vendor (may be different from original vendor — repair specialists), Estimated Cost, Loaner Equipment (if needed: issue from inventory PR-07 as loaner, track return when primary is repaired). Cost tracking: repair cost vs replacement cost analysis helps determine when to stop repairing and replace.
ZVE PROS REPAIR wrapper -> repair data within prosthetics files. Per Prosthetics Basic UM: Suspense Menu (SU) for tracking in-process items.
LOANER EQUIPMENT is critical for ambulatory aids and prosthetic limbs: a veteran cannot be without their wheelchair or prosthetic leg for weeks while the primary device is repaired. The system tracks loaner issue and return, ensuring loaners are recovered when the primary device is returned. Repair-vs-Replace Decision: the system can display cumulative repair costs for an item alongside the replacement cost. If cumulative repairs exceed a configurable percentage of replacement cost (e.g. 60%), system suggests replacement may be more cost-effective. Per NPPD: repair activities are tracked separately from new activities on NPPD worksheets, enabling analysis of repair frequency and cost by item category.
PR-10
Recall Management
Worklist + Detail View
Manufacturer device recalls affecting prosthetic items. VistA Evolved ADDITION (not in legacy VistA Prosthetics). When a manufacturer issues a recall (from FDA recall database or manufacturer notification): Recall Entry form (device manufacturer, model, affected serial/lot range, recall classification: Class I/II/III, hazard description, recommended action: repair/replace/return/monitoring). Affected Patient Identification: system searches PR-02 patient records and PR-07 inventory for items matching the recalled manufacturer + model + serial/lot range. Results: list of affected veterans with: name, item, serial number, issue date, current status. Recall Response Documentation: per affected veteran: response action taken (device replaced, device repaired, veteran contacted for monitoring, veteran declined intervention), response date, follow-up plan. Notification: auto-generate patient notification letters for affected veterans. Similar pattern to Surgery SG-12 Implant Registry but for prosthetic devices.
ZVE PROS RECALL wrapper -> recall tracking data (VE-specific), cross-referenced with PROSTHETICS #669 (patient equipment by manufacturer/model/serial) and PIP inventory (stock items by manufacturer/model/serial).
FDA RECALL RESPONSE: Class I recalls (reasonable probability of serious health consequences or death) require URGENT response — all affected patients identified and contacted within 24-48 hours. The system generates a prioritized action list. Class II (may cause temporary or medically reversible adverse health consequences) and Class III (unlikely to cause adverse health consequences) allow more time but still require documented response for every affected item. PROACTIVE MONITORING: the system can be configured to auto-check FDA recall database (if API access available) against the facility's prosthetic equipment inventory and patient records, auto-generating alerts when matches are found. This replaces the manual process of hearing about a recall and then searching records.
PR-11
Vendor Management
Worklist + Detail View + Form
Vendor profiles and contract management. Vendor table: Vendor Name, Contact, Address, Phone, Vendor Type (Commercial/VA Contract/BPA-Mandatory/BPA-Optional), Contract# (if applicable), Contract Expiration, Performance Rating (1-5 based on delivery timeliness, product quality, service responsiveness), Volume (total POs issued this year, total $ spent). Per Prosthetics GUI UM: 'Enter/Edit Contract# (EC)' for contract management, 'Enter Waiver or Excluded Notice (EW)' for non-contract procurement documentation. Vendor detail: full profile + order history + performance metrics + contract terms + pricing schedule. BPA/Mandatory Contract compliance: system tracks whether items on mandatory national contracts are being purchased from the contracted vendor. Non-compliance (purchasing from non-contract vendor without waiver) flagged for Prosthetic Service Chief review.
ZVE PROS VENDOR wrapper -> vendor data within prosthetics files + IFCAP vendor file #440.
BPA (Blanket Purchase Agreement) and Mandatory Contract compliance is a national Prosthetics program priority. PSAS (Prosthetic and Sensory Aids Service) at VA Central Office negotiates national contracts for high-volume items (wheelchairs, hearing aids, prosthetic components) to achieve volume pricing. Facilities are expected to use these contracts. Per Prosthetics GUI UM: 'Excluded means that an Item (HCPCS) issued is not covered by a National Prosthetics Mandatory BPA/Contract' — this is acceptable. But using a non-contract vendor for a mandatory-contract item without a waiver is a compliance issue. The vendor management page tracks this compliance.
PR-12
Prosthetics Reports
Report (WF-01 5.5)
Reports: (1) Expenditure by HCPCS Category (wheelchairs, prosthetic limbs, hearing aids, orthotics, home O2, DME, sensory aids — by period, by vendor). (2) Utilization by Patient Demographics (items issued by age, gender, era, service connection status). (3) Turnaround Time (consult received to item delivered — by item category). (4) Vendor Performance (delivery timeliness, product quality complaints, return rates). (5) Budget Utilization (obligations + expenditures vs allocation by fund control point, projected year-end position). (6) NPPD Summary (NPPD Group/Line totals matching national reports for local verification). (7) Replacement Schedule Compliance (% of items replaced on schedule vs overdue). (8) Home Oxygen Cost Analysis (cost per patient per month, comparison across vendors, opportunities for modality optimization). (9) Repair-to-Replacement Ratio (repair frequency and cost vs replacement cost by item category). (10) HISA Grant Summary (grants awarded, remaining lifetime eligibility, average grant amount, most common modifications).
ZVE PROS REPORTS wrapper -> aggregated from PROSTHETICS #669, PURCHASE ORDER #668, IFCAP #442, NPPD data, PIP inventory, RMPO oxygen data.
Standard Report template. KEY METRIC: Turnaround Time (consult to delivery) is the primary access metric for Prosthetics — equivalent to appointment wait time for clinic access. VA monitors prosthetic delivery timeliness nationally. Target: routine items delivered within 30 days, urgent within 5 business days. NPPD Summary for local verification: facilities should compare their local NPPD summary against the national NPPD report (returned from Hines) to verify data consistency. Discrepancies may indicate: local data correction after submission, transmission errors, or timing differences.
PR-13
Prosthetics Configuration
Config (WF-01 5.6)
Configuration sections: (1) Site Parameters (per Prosthetics Basic UM: 'SP Enter/Edit Site Parameters' in Utilities menu — station name, division, contact information). (2) HCPCS Code Tables (PSAS HCPCS master file — updated nationally by PSAS, with local synonym management per NPPD UM 'HCPCS Synonyms AE option'). (3) PROS ITEM MASTER (local item catalog mapping items to HCPCS codes, vendors, costs). (4) Approval Authority Tiers (cost thresholds for PO approval: e.g. <$2,500 Prosthetic Rep auto-approve, $2,500-$25,000 Service Chief, >$25,000 Facility Director or Chief of Staff). (5) Vendor Setup (add/edit vendor profiles, link to IFCAP vendor file #440, BPA/contract assignment). (6) Replacement Schedule Definitions (per item category: expected service life, lead time for reorder, proactive outreach window). (7) NPPD Parameters (reporting periods, submission schedule, HCPCS-to-Group/Line mapping configuration — per NPPD: 'Prosthetic HCPCS are mapped to the NPPD Groups/Lines'). (8) Home Oxygen Parameters (vendor assignments, delivery schedules, renewal reminder intervals, qualifying criteria thresholds). (9) Inventory Parameters (PIP: barcode label format for Zebra printer per PIP documentation, storage locations, physical count schedule, par levels). (10) Security Key Assignments (RMPRMANAGER: full prosthetics management access, RMPRSUPERVISOR: supervisory functions, standard user keys per Prosthetics TM: 'The following security keys are exported with Prosthetics').
ZVE PROS CONFIG wrapper -> Prosthetic Site Parameter file, PSAS HCPCS tables, PROS ITEM MASTER, IFCAP configuration, PIP configuration.
CAUTION banner. HCPCS code table updates: PSAS national office distributes HCPCS updates periodically (new codes, retired codes, description changes). The system must apply these updates to maintain NPPD reporting accuracy. Approval authority changes: affect purchasing workflow for ALL prosthetic purchases — require Prosthetic Service Chief and Fiscal Service coordination. Replacement schedule changes: affect proactive outreach timing for all patients with the affected item category — changes should be clinically validated before implementation.
