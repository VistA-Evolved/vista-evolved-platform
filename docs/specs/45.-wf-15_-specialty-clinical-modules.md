# PART A: Women's Health (7 Pages)
Women's Health provides gender-specific preventive care tracking for the 800,000+ women veterans served by VA. The WV namespace stores all data in the ^WV global across files #790-794. Per WH UM: 'Patient Management manages individual patient care: procedures, due dates, and correspondence. Possible to maintain date of next PAP smear, colposcopy, or mammogram, patient current PAP regimen.' WH integrates with Clinical Reminders (PXRM) for automated screening due-date management, and with Radiology (RA) for automatic mammogram import.

Accessed as a sub-module within the Clinical Workspace (WF-02) under a 'Women's Health' tab, or as a standalone workspace for sites where WH has dedicated staff (configurable per Tenant Profile). Uses HORIZONTAL NESTED TABS: Profile, Breast Screening, Cervical Screening, Maternity, Notifications, Reports, Config.

WH-02
Breast Screening Management
Worklist (WF-01 5.1) + Detail View (WF-01 5.2)
PATIENT-LEVEL VIEW (when patient selected): Mammography history table: Date, Exam Type (Screening/Diagnostic per BI-RADS classification), BI-RADS Assessment (0-6 with color badges per Design System semantic colors and ACR standard language: 0='Incomplete-Needs Additional Imaging' amber, 1='Negative' green, 2='Benign Finding' green, 3='Probably Benign-Short Interval Follow-Up Recommended' yellow, 4='Suspicious Abnormality' orange with sub-categories 4A/4B/4C, 5='Highly Suggestive of Malignancy' red, 6='Known Biopsy-Proven Malignancy' dark-red), Recommendation (from radiology report), Follow-Up Status (pending additional imaging / biopsy scheduled / biopsy complete / pathology received), Notification Status (patient letter sent Y/N per MQSA). Treatment Need (auto-populated from #790: Normal Screening / Needs Treatment / Treatment Active / Treatment Complete) + Treatment Due Date (auto-calculated per screening interval: annually for age 45-54, every 2 years for 55+ per VHA guidelines, or shorter intervals based on BI-RADS 3 follow-up recommendation). ABNORMAL FOLLOW-UP WORKFLOW: for BI-RADS 0/4/5: system creates a follow-up tracking entry requiring: additional imaging ordered -> biopsy referral if indicated -> biopsy result -> pathology result -> treatment plan -> outcome. Each step tracked with date/status. Days since abnormal result prominently displayed (per VHA: mammography results communicated within 30 days per MQSA, BI-RADS 4/5 'Suspicious/Highly Suggestive of Malignancy results within 3-5 days'). POPULATION-LEVEL VIEW (no patient selected): Worklist of all WH patients: columns: Patient, Last Mammogram Date, BI-RADS, Next Due Date, Status (Current=green, Due=amber, Overdue=red, Abnormal Follow-Up Active=red-pulse), MCC/Case Manager. Filter: screening status (Due/Overdue/Abnormal Follow-Up), provider panel, date range. This population view enables the WH coordinator to identify all patients needing outreach.
WV PATIENT #790 (breast treatment need, breast due date), WV PROCEDURE sub-files (procedure history), Radiology results (auto-imported mammograms with BI-RADS from RA*5.0*2 integration), WV NOTIFICATION PURPOSE #790.404 (letter tracking) — via ZVE WH BREAST wrapper.
BI-RADS from RADIOLOGY INTEGRATION: per WH UM, mammography results are 'automatically imported and stored from the Radiology/Nuclear Medicine package' when site parameter 'Import Mammograms from Radiology' is enabled. The BI-RADS assessment category is captured from the radiology report. Per VHA directive: 'BI-RADS assessment category must be entered and associated to a radiology order in CPRS. Systems for tracking and management of mammography and breast cancer will not operate accurately without BI-RADS entered into CPRS.' VistA Evolved validates that every imported mammogram has a BI-RADS code; imports without BI-RADS are flagged for Radiology to complete. MQSA COMPLIANCE: per MQSA (Mammography Quality Standards Act, FDA-regulated): patient notification letter must be sent within 30 days of examination. The system tracks: exam date, letter generation date, letter send date. Overdue letters (>30 days without send) are flagged RED on the Notification Management page (WH-05). CROSS-WORKSPACE LINK TO RADIOLOGY RA-06: the WH breast screening page displays the SAME BI-RADS data that appears on the Radiology mammography page (RA-06). Both pages reference the same underlying data — changes in one are reflected in the other. The WH view adds screening-specific tracking (due dates, treatment needs, population-level management) that the Radiology view does not provide.
WH-03
Cervical Screening Management
Worklist + Detail View
PATIENT-LEVEL: Cervical screening history table: Date, Test Type (Pap only / HPV co-test / HPV primary), Pap Result (per Bethesda System classification: NILM=green, ASC-US=yellow, ASC-H=orange, LSIL=yellow, HSIL=red, AGC=orange, AIS=red, SCC=red, Adenocarcinoma=red), HPV Result (Negative=green, Positive=red, with high-risk type identification if available), Combined Interpretation (per ASCCP risk-based management guidelines: recommended follow-up action based on Pap + HPV combination + history), Follow-Up Status (for abnormal: colposcopy referral -> colposcopy performed -> biopsy results -> treatment if needed). Cervical Treatment Need (from #790 cervical treatment need field) + Treatment Due Date (auto-calculated per guidelines: 3 years for Pap-only, 5 years for co-testing, shorter for abnormal follow-up per ASCCP management algorithms). 7-DAY ABNORMAL COMMUNICATION RULE: per VHA directive, 'abnormal cervical pathology report, the results must be communicated within 7-calendar days of the report becoming available to the ordering provider.' System tracks: result available date, provider notification date, patient notification date. >7 days without provider notification = RED alert. POPULATION-LEVEL: Worklist of patients due/overdue for cervical screening, same pattern as WH-02 population view. CYTOPATHOLOGY INTEGRATION: Pap results auto-populated from AP Cytology (AP-06 from WF-04 Lab) when cytopathology report is signed out. Bethesda classification mapped from AP cytology diagnostic codes.
WV PATIENT #790 (cervical treatment need, cervical due date), AP Cytopathology results (Bethesda from AP-06), WV NOTIFICATION PURPOSE #790.404 — via ZVE WH CERVICAL wrapper.
ASCCP GUIDELINE ENGINE: the system implements ASCCP (American Society for Colposcopy and Cervical Pathology) risk-based management guidelines as a decision-support engine. Based on: current Pap result + current HPV result + prior screening history + patient age, the system RECOMMENDS the appropriate follow-up action (repeat screening in N years, colposcopy referral, treatment). This removes the burden of memorizing complex management algorithms from the clinician. The recommendation is displayed prominently with the result. Clinician can accept the recommendation (system auto-generates appropriate follow-up order/referral) or override with documented clinical reasoning. HPV VACCINATION TRACKING: for younger patients, HPV vaccination status displayed (from Immunization data V IMMUNIZATION). Vaccination affects future screening recommendations.
WH-04
Maternity Care Coordination
Detail View + Form
Pregnancy management: (1) PREGNANCY REGISTRATION: EDD entry (from dating ultrasound or LMP calculation), gestational age (auto-calculated, displayed as weeks+days), trimester indicator, high-risk flag (for pregnancies with: advanced maternal age >35, gestational diabetes, pre-eclampsia, multiple gestation, prior pregnancy complications). (2) MCC ASSIGNMENT: Maternity Care Coordinator assignment from WV Case Manager #790.01. Per WH UM: 'Maternity care coordinators assigned to pregnant female patients. MCC data stored in WV Case Manager #790.01.' MCC serves as the veteran's primary coordinator throughout pregnancy and postpartum period, connecting VA care with community maternity care. (3) PRENATAL VISIT LOG: most maternity care provided by non-VA community OB providers (per VHA: 'Maternity care is provided by non-VA providers in non-VA facilities'). Visit tracking: date, provider (community OB name), visit type (routine prenatal, ultrasound, specialty consultation), gestational age at visit, findings/notes. MCC documents community provider visits to maintain VA awareness of pregnancy progress. (4) PRENATAL LABS: tracking grid showing required prenatal labs with status: blood type/Rh (from BB-02), CBC, rubella titer, hepatitis B surface antigen, syphilis screening, HIV, urine culture, glucose challenge test (24-28 weeks), GBS culture (35-37 weeks), Rh antibody screen (if Rh-negative). Each: Due Date, Status (Complete/Pending/Overdue), Result link. (5) DELIVERY OUTCOME: date, delivery method (vaginal/cesarean), complications, newborn status, facility. (6) POSTPARTUM: 6-week postpartum visit tracking, contraception counseling, screening resumption scheduling.
WV PATIENT #790 (pregnancy status, EDD), WV Case Manager #790.01 (MCC assignment), Lab #60 (prenatal labs), Blood Bank BB-02 (blood type), Community Care tracking data — via ZVE WH MATERNITY wrapper.
COMMUNITY CARE COORDINATION is the defining challenge: VA covers maternity care but most of it is provided OUTSIDE VA facilities by community OB providers. The MCC's role is to be the bridge: ensuring the veteran gets timely prenatal care from the community provider, that lab results are ordered by VA and results shared with the community provider, that complications are escalated to VA oversight, and that postpartum follow-up brings the veteran back to VA for ongoing primary care. This page is the MCC's PRIMARY TOOL for managing their caseload of pregnant veterans. PRENATAL LAB TRACKING is a patient safety feature: certain prenatal labs are time-sensitive (GBS culture must be at 35-37 weeks to determine antibiotics needed during delivery). The system calculates when each lab is due based on EDD and flags overdue labs. Because the veteran may be receiving care from a community provider who orders their own labs, the MCC must verify whether labs were done at VA or externally — if externally, results should be obtained and documented.

MQSA 30-DAY LETTER MANDATE: per MQSA, every mammography patient must receive a lay-language results letter within 30 days. This is a FEDERAL (FDA) regulatory requirement, not just VA policy. The system tracks this deadline per patient per exam and flags any exam >30 days without letter sent. Per VHA directive: 'FDA-approved language corresponding to the appropriate BI-RADS assessment category must be included in the impression.' The letter templates include lay-language BI-RADS explanations (e.g. BI-RADS 1: 'Your mammogram showed no signs of cancer. Continue routine screening.', BI-RADS 0: 'Your mammogram showed an area that needs additional pictures. This does not mean you have cancer. Please call to schedule additional imaging.'). MULTI-CHANNEL NOTIFICATION: legacy VistA WH only supports printed letters. VistA Evolved adds: My HealtheVet secure message (if patient has MHV account), SMS text (if phone on file + patient consented to text), email (if email on file + patient consented). Channel preference stored per patient. Print remains the default/fallback for patients without electronic preferences.

(6) CANCER DETECTION: BI-RADS 4/5 positive predictive value (PPV: % of biopsied suspicious findings confirmed malignant), cervical cancer detection rates. Standard Report template. KEY METRICS FOR VHA WH PROGRAM: screening compliance rates feed national Women's Health program reporting. Per VHA directive: facilities must monitor screening rates and report to VISN and national WH program office. Cancer detection rates and PPV are quality metrics for the mammography program (too-high recall rate suggests over-calling; too-low PPV suggests unnecessary biopsies). HEDIS-LIKE ALIGNMENT: breast cancer screening compliance aligns with HEDIS BCS measure (Breast Cancer Screening) and SAIL prevention composite. # PART B: Oncology / Tumor Registry (8 Pages)

ONC-01
Cancer Suspense Worklist
Worklist (WF-01 5.1)
Cases identified by automatic case finding awaiting registrar review. Columns: Patient (name + SSN last 4 + DOB), Source (icon + text: Pathology=microscope icon per AP surgical pathology finding, Radiology=imaging icon per radiology report suggesting malignancy, PTF=hospital icon per Patient Treatment File discharge diagnosis with cancer ICD code — per ONC UM: 'Automatic Case Finding searches pathology (CF), radiology (RA) and Patient Treatment File (PT) for eligible cases'), Organ/Tissue (anatomic site from source document), Lab Morphology (histologic type from pathology report if source is AP), Suspense Date (date case was identified — per ONC UM: 'Having a date in the Suspense Date field in the Oncology Patient file #160'), Admission Date, Discharge Date. Filter: Source type (Path/Rad/PTF/Manual), Date range, Accession status (Not Yet Accessioned / Accessioned / Deleted). Sort: default by Suspense Date (oldest first — FIFO processing). Actions: 'Accession' (accepts case as reportable — assigns accession number, moves to abstracting queue, per ONC UM: 'cases held in Suspense until accessioned for abstracting'), 'Delete' (case reviewed and determined NOT reportable — per ONC UM: 'manually delete cases; refer to Deleting a VA Patient from Suspense'), 'Flag for Review' (uncertain reportability — needs supervisor or Cancer Committee review). Per ONC UM: 'Suspense file list — printout lists patients according to source (Surgical Pathology, Cytopathology, Electron Microscopy, Autopsy, PTF, Radiology, or manual entry) then in order of suspense date.'
Oncology Patient #160 (Suspense Date field), automatic case finding results from: AP Surgical Pathology (LR #60 sub-files), Radiology reports (RAD/NUC MED), PTF #45 (discharge diagnoses) — via ZVE ONC SUSPENSE wrapper.
AUTOMATIC CASE FINDING is the FOUNDATIONAL feature of the tumor registry. Without it, cancer cases would be missed. Per ONC UM: 'Enter a range, start date, and end date, and the computer searches pathology (CF), radiology (RA) and the Patient Treatment File (PT) for eligible cases in that date range. Each search is run separately. Cases meeting the defined criteria are captured electronically and added to Suspense.' VistA Evolved ENHANCEMENT: in addition to the three legacy sources, VistA Evolved adds NLP (Natural Language Processing) scanning of clinical notes (TIU #8925) for cancer-suggestive language (e.g. 'biopsy-proven adenocarcinoma', 'metastatic disease', 'staging CT shows mass') — flagged cases added to Suspense with 'NLP' source tag for registrar review. This captures cases that may not yet have pathology confirmation but have clinical documentation of cancer. TIMELINESS: ACoS Commission on Cancer requires that cases be accessioned within defined timeframes after diagnosis. The Suspense Worklist shows aging (days since suspense date) with amber >30 days and red >90 days to drive timely processing.
ONC-02
Cancer Abstract Entry (OncoTrax)
Form (WF-01 5.3) COMPLEX MULTI-SECTION
The comprehensive cancer abstract — per ONC UM: 'An abstract is a summary of pertinent information about the patient, the cancer, the treatment, and the outcome. Components include patient demographic information, cancer identification, extent of disease, stage at diagnosis, first course of treatment, recurrence, and subsequent therapies or progression.' Organized as a MULTI-SCREEN FORM (modernizing OncoTrax's prompt-driven terminal interface per ONC UM: 'AS Abstract Screens Menu (80c)'). Sections: (1) DEMOGRAPHICS (auto-populated from PATIENT #2: name, SSN, DOB, sex, race, ethnicity, address, vital status). (2) CANCER IDENTIFICATION: Primary Site (ICD-O-3 Topography code — searchable picker showing anatomic sites. Per ONC TM: 'ICDO Topography file creates site-specific topography coding per SEER.' Once topography selected, all subsequent prompts are site-specific per ONC UM: 'OncoTrax is prompt driven. Once a specific Topography Code is selected, all successive prompts displayed are specific.'), Histologic Type (ICD-O-3 Morphology code — per ONC TM: 'ICDO Morphology file'), Behavior Code (/0 benign, /1 borderline, /2 in situ, /3 malignant — per ICD-O-3), Grade/Differentiation, Laterality, Diagnostic Confirmation Method, Date of Diagnosis. (3) EXTENT OF DISEASE: tumor size, extension, lymph nodes examined + positive, distant metastasis sites. (4) STAGING: via ONC-03 Staging Workstation (embedded or linked). (5) FIRST COURSE TREATMENT: Surgery (procedure type, date, surgical approach, margins), Radiation (modality, dose, target, dates), Chemotherapy (agents via ONC-06 chemo drug, dates, intent: curative/adjuvant/palliative), Hormonal Therapy, Immunotherapy/Targeted Therapy, Other treatment. Per ONC UM: 'CHEM Enter/Edit chemotherapeutic Drug File.' (6) FOLLOW-UP: per ONC-05. (7) BIOMARKERS (for applicable sites): ER/PR/HER2 (breast), MSI/MMR (colorectal), PD-L1 (lung), BRCA (breast/ovarian), PSA/Gleason (prostate per ONC TM: 'Gleason Patterns Pathological fields'), etc.
Oncology Patient #160, Oncology Primary #165.5, Oncology Contact #165 — via ZVE ONC ABSTRACT wrapper. Cross-references: PATIENT #2 (demographics), AP reports (pathology), Surgery #130 (surgical treatment), Pharmacy #52 (chemotherapy), Radiology (radiation treatment records).
MOST STRUCTURED FORM IN THE ENTIRE PRODUCT: hundreds of NAACCR data elements across dozens of fields. The abstract must be complete enough to satisfy: ACoS Commission on Cancer accreditation standards, VACCR (VA Central Cancer Registry) reporting requirements, state cancer registry reporting requirements, and NAACCR data quality standards. Per ONC UM: 'An abstract must be completed for all cases that meet the criteria for inclusion in the registry. The standards are set forth by the American College of Surgeons and VACCR reportable criteria.' SITE-SPECIFIC PROMPTING is key to usability: rather than presenting all possible fields for all cancer types, OncoTrax shows ONLY the fields relevant to the selected primary site. A breast cancer abstract shows ER/PR/HER2 fields; a prostate abstract shows PSA/Gleason fields; a lung abstract shows PD-L1 fields. This reduces cognitive load and prevents irrelevant data entry. AUTO-POPULATION FROM CROSS-WORKSPACE DATA: VistA Evolved auto-populates abstract fields from other workspaces where data already exists: demographics from PATIENT #2, pathology findings from AP reports (AP-05), surgical details from Surgery operative reports (SG-05), chemotherapy agents from Pharmacy medication records, radiation details from Radiology treatment records. The registrar reviews and validates auto-populated data rather than re-entering it manually — dramatically reducing abstracting time. NAACCR VALIDATION: before the abstract can be 'completed,' the system runs NAACCR edit checks (per ONC TM: NAACCR version compliance) validating: required fields are populated, code values are valid for the selected site, logical consistency (e.g. tumor size cannot exceed anatomic site limits, positive lymph nodes cannot exceed examined lymph nodes).

Form (WF-01 5.3)
AJCC TNM staging entry and validation. Layout: Left 50%: staging entry form. Clinical Stage: cT (tumor size/extent), cN (regional lymph node involvement), cM (distant metastasis) — each a dropdown with options SPECIFIC to the selected primary site per AJCC 8th edition staging manual (per ONC TM: 'AJCC Staging Groups file, AJCC TNM coding schemes'). Pathological Stage: pT, pN, pM — from surgical pathology findings post-operatively. Post-Therapy Stage: ypTNM — if neoadjuvant therapy (chemo/radiation before surgery). Stage Group: AUTO-CALCULATED from T+N+M combination per AJCC 8th edition grouping tables. Display: large stage group badge (e.g. 'Stage IIB' in --font-size-2xl with color: Stage 0/I=green, Stage II=yellow, Stage III=orange, Stage IV=red). Right 50%: staging reference panel showing: AJCC staging criteria for the selected primary site (what constitutes T1 vs T2 vs T3 for this tumor type), prognostic factors (grade, biomarkers that affect stage group for certain sites per AJCC 8th ed prognostic staging). Per ONC TM: 'Check Validity of TNM Fields' option validates staging. Summary Stage: SEER Summary Stage assignment (localized/regional/distant) in addition to AJCC TNM. Site-Specific Factors: additional data elements required for certain sites by NAACCR/SEER (e.g. Breslow thickness for melanoma, mitotic rate for GIST).
ZVE ONC STAGING wrapper -> AJCC Staging Groups file within ONC package, Oncology Primary #165.5 (staging fields).

# PART C: Dental (8 Pages)
VA Dental Service provides comprehensive oral healthcare per VHA Directive 1130. Dental uses ADA CDT (Current Dental Terminology) codes for procedure coding — distinct from CPT codes used by medical services. CDT codes range D0100-D9999 and are updated annually by the ADA. DEN-02
Periodontal Chart
Detail View SPECIALIZED
PERIODONTAL EXAMINATION GRID: the most measurement-intensive display in the entire product. Layout: Horizontal grid with teeth as columns (1-16 upper, 17-32 lower) and measurement rows. Per tooth: 6 PROBING DEPTH SITES (Mesiobuccal, Buccal, Distobuccal on facial aspect; Mesiolingual, Lingual, Distolingual on lingual aspect). Each site: numeric value in mm (typically 1-12mm). COLOR CODING PER DEPTH: 1-3mm = GREEN (healthy sulcus), 4-5mm = YELLOW (moderate pocketing — needs monitoring or treatment), 6+mm = RED (deep pocketing — indicates significant periodontal disease). PROBING DEPTH BAR CHART: below the numeric grid, a visual bar chart showing probing depth around each tooth as colored bars — provides instant visual of pocket depth distribution. Per VHA Directive 1130: 'Periodontal Screening Record (PSR) indicates the need per ADA standard.' PSR SCORE: summary screening score (Code 0-4 + * for furcation) calculated from deepest probing depth per sextant. HISTORICAL COMPARISON: toggle to overlay prior periodontal exam data — current values shown solid, prior values shown as dotted outline. Improvement (shallower pockets) shown in GREEN delta, worsening (deeper pockets) shown in RED delta.
ZVE DEN PERIO wrapper -> periodontal measurement data structures storing 192 probing depth values + BOP + recession + furcation + mobility per tooth.
192 MEASUREMENTS: 32 teeth x 6 probing sites = 192 individual probing depth measurements per periodontal exam. QUICK-ENTRY MODE: for efficient data entry during the examination, the system provides a sequential entry mode: the hygienist calls out probing depths in order (e.g. '3-2-3-4-3-3' for one tooth's six sites) and the values are entered sequentially, advancing to the next site/tooth automatically. Voice entry option: if speech-to-text is available, the hygienist can dictate probing depths hands-free (critical because they are holding the periodontal probe in one hand and retracting tissue with the other). CLINICAL ATTACHMENT LEVEL (CAL) is the gold standard for measuring periodontal disease severity. CAL = probing depth + recession. A tooth with 5mm probing depth and 2mm recession has 7mm attachment loss (indicating more disease than the probing depth alone would suggest). The system auto-calculates CAL from the entered probing depth and recession values. TREATMENT PLANNING INTEGRATION: periodontal chart data directly informs DEN-03 treatment planning. Sites with >=4mm pocketing with BOP are candidates for scaling and root planing (SRP). Sites with >=6mm pocketing may need periodontal surgery. The system can auto-suggest treatment based on charted findings.

CAUTION banner. ANNUAL CDT UPDATE: the most critical configuration event is the annual CDT code update each January. New codes are added, existing codes may be revised or deleted. Per Vol 17C Rule 5: 'CDT codes updated annually. The CDT code table must be updatable independently.' The update process: (1) obtain new CDT data file from ADA, (2) import into staging, (3) review changes (new codes, deleted codes, revised descriptions), (4) activate for new date-of-service, (5) maintain prior year codes for historical encounters. TOOTH NUMBERING is a configuration decision with significant clinical impact: the ADA Universal system (1-32) is standard in the US, but the FDI system (11-48 two-digit) is standard internationally. For international VistA Evolved deployments, the tooth numbering system must match local dental conventions. Switching numbering systems on an existing system requires careful data migration.
