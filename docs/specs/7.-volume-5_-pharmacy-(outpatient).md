# 1. Foundation: Why Pharmacy Is the Hardest Workspace
Pharmacy is fundamentally different from Scheduling (which has 150+ RPCs) and Clinical Workspace (which has hundreds of CPRS RPCs). Outpatient Pharmacy has VERY LIMITED RPC coverage. Almost all pharmacist and technician workflows are terminal-based, driven through the PSO namespace menus with ListManager-style screens. The package contains approximately 850 routines but exposes very few via RPC Broker.
This means: every major pharmacy fulfillment workflow (verification, dispensing, suspense management, refill processing, controlled substance handling, label printing) requires CUSTOM WRAPPER RPCs built in the ZVE RX namespace.
Orders originating from CPRS (clinician places medication order via ORWDX SAVE) flow into PENDING OUTPATIENT ORDERS #52.41 and are then processed by the pharmacist through the terminal option [PSO LM BACKDOOR ORDERS] Patient Prescription Processing. The modern pharmacy workspace must replicate this flow.
Source: PSO V7.0 Technical Manual (June 2020): 'The Outpatient Pharmacy V.7.0 package provides a method for managing the medications given to veterans who have visited a clinic or who have received prescriptions upon discharge from the hospital. Prescription labels are automatically generated and refill request forms are printed. Medication histories are kept online to permit checks for potential interactions.'

## 4.4 RX-03: Verify Prescription
### Purpose
Pharmacist verification of a prescription. Verification is a clinical safety check — the pharmacist confirms the prescription is correct, safe, and appropriate for the patient before it can be dispensed. This is a legally significant action.
Source: PSO TM: RX VERIFY #52.4 holds prescriptions awaiting verification. Terminal: Verification menu [PSO VERIFY]. The verification process checks the prescription against the patient's allergy history, other active medications, and clinical appropriateness.

CRITICAL DEVELOPMENT NOTE: Every custom wrapper must be studied against the corresponding PSO namespace routines BEFORE implementation. The PSO package has complex business rules embedded in its M routines (drug interaction checking, allergy cross-referencing, order check firing, copay calculation, CMOP eligibility, CS accountability, label formatting, ECME claims processing). The wrappers must invoke or replicate these business rules, NOT bypass them. A wrapper that writes directly to #52 without running PSO business logic will produce incorrect prescriptions.

# 6. Rules for Design and Engineering
Rule 1: Patient allergies must be visible at ALL times during pharmacy work. They cannot be hidden, collapsed, or dismissed. This is a patient safety requirement.
Rule 2: All 9 prescription statuses must be represented: Active, Non-Verified, Hold, Provider Hold, Suspended, Discontinued, Discontinued (Edit), Expired, Deleted. Do not simplify to Active/Inactive.
Rule 3: Controlled substances (Schedule II-V) require separate handling. The PSDRPH key must be enforced. CS prescriptions must be visually distinct from non-CS prescriptions.
Rule 4: Drug interaction and allergy warnings must be displayed prominently and cannot be auto-dismissed. Pharmacist must explicitly acknowledge with documented override reason.
Rule 5: Verification is a legally significant action. Only pharmacists with the PSO key can verify. Technicians can view and prepare but NEVER verify.
Rule 6: Prescription barcode format is NNN-NNNNNNN where first 3 digits = station number. The UI must support barcode scanning for patient/prescription lookup.
Rule 7: Outpatient and Inpatient pharmacy are SEPARATE packages with separate files and workflows. This volume covers outpatient only (PSO namespace, PRESCRIPTION #52). Inpatient Medications (PSJ namespace, files #55.06 and others) requires its own volume.
Rule 8: Custom wrappers must invoke PSO business logic, not bypass it. Direct writes to #52 without running interaction checks, copay calculations, and order completion logic will produce incorrect prescriptions.
Rule 9: Background jobs (Autocancel on Admission, Nightly Rx Cost Compile, Expire Prescriptions, Auto-delete from Suspense, SPMP Export) must continue running via TaskManager. The modern UI should provide a monitoring view (RX-10) but NOT attempt to replace TaskManager scheduling.
Rule 10: The OneVA Pharmacy flag in #59.7 controls whether this VistA instance participates in cross-site prescription sharing. The modern UI must respect this flag and display OneVA Pharmacy prescriptions appropriately when enabled.
Rule 11: ePrescribing (inbound prescriptions from external providers via ERX HOLDING QUEUE #52.49) is a separate workflow requiring PSO ERX TECH or PSO ERX ADV TECH keys. It is deferred to Wave 3+.
Rule 12: Label printing is essential pharmacy infrastructure. Labels must include barcodes in the station-sequence format. Laser and dot-matrix printers are both used. The modern UI must support configurable label formats per OUTPATIENT SITE #59 parameters.
