# 1. Foundation: Why Inpatient Pharmacy Is a Separate System
Inpatient Medications (PSJ namespace) is a completely separate VistA package from Outpatient Pharmacy (PSO namespace, covered in Volume 5). They share some infrastructure (DRUG file #50, PHARMACY PATIENT file #55) but have different workflows, different user roles, different terminal menus, different security keys, and different documentation sets.
Key differences from Outpatient Pharmacy (Volume 5):
(a) LOCATION: Outpatient = pharmacy window dispensing to ambulatory patients. Inpatient = ward-based medication management for admitted patients.
(b) ORDER MODEL: Outpatient = PRESCRIPTION #52 (single medication, refillable). Inpatient = orders stored in PHARMACY PATIENT #55 UNIT DOSE sub-file #55.06 or IV sub-file #100 (scheduled, continuous, one-time, PRN — not refillable).
(c) DISPENSING: Outpatient = fill at window, patient takes home. Inpatient = unit dose packages delivered to ward, nurse administers to patient at bedside.
(d) ADMINISTRATION: Outpatient = patient self-administers. Inpatient = nurse administers, documents on MAR (Medication Administration Record), scans with BCMA.
(e) USER ROLES: Outpatient = pharmacist + technician. Inpatient = pharmacist + nurse + pharmacy technician + supervisor, each with different security keys and different views.
(f) IV MEDICATIONS: Outpatient pharmacy rarely handles IV. Inpatient has an entire IV Medications module for admixture preparation, rate calculations, compatibility checking, and bag labeling.
Source: PSJ TM: 'PHARMACY PATIENT file (#55) for Unit Dose orders in the UNIT DOSE sub-file (#55.06), or the IV sub-file (#100).' Pharmacist UM: Three user manuals (Pharmacist, Nurse, Supervisor) reflect three distinct user roles. Supervisor UM: 'Users who do not hold any of these keys are seen as ward clerks.'

# 5. Rules for Design and Engineering
Rule 1: Inpatient Medications and Outpatient Pharmacy are SEPARATE systems with separate workflows, menus, security keys, and data structures. They share DRUG #50 and PHARMACY PATIENT #55 but use different sub-files within #55.
Rule 2: Order verification by a pharmacist (PSJ RPHARM key) is REQUIRED before any inpatient medication can be administered. Unverified orders must NOT appear as administrable in MAR or BCMA.
Rule 3: Three user roles (Pharmacist, Nurse, Pharmacy Tech) see different views and have different action permissions. The modern UI must enforce role-based access via security key checks.
Rule 4: Ward groups and clinic groups organize order processing. Orders are NOT just a flat queue — they are organized by ward/clinic group for efficient pharmacist batch processing.
Rule 5: Order checks (allergy, interaction, duplicate, dosing) must run DURING order entry, not after. The pharmacist must review and resolve all checks before the order can be completed.
Rule 6: IV orders have a multi-component structure: base solution(s) + additive(s) + rate. This is NOT a single-drug order like Unit Dose. The IV order form must support multiple solutions and multiple additives per order.
Rule 7: Stop dates are auto-calculated from WARD PARAMETERS #59.6 or SYSTEM PARAMETERS #59.7. Maximum 367 days. Warning if start date exceeds 7 days from login date. The modern UI must implement these calculations and warnings.
Rule 8: IMO (Inpatient Medications for Outpatients) orders use CLINIC DEFINITION #53.46 parameters. If a clinic is not defined in #53.46, the order will not display in BCMA. The modern UI must enforce clinic definition before allowing IMO orders.
Rule 9: Patient allergy assessment is a prerequisite for order entry. If no allergy data exists, the system must force entry of allergy information or explicit 'No Known Allergies' confirmation before allowing medication orders.
Rule 10: When a patient transfers between wards, medication orders may be automatically discontinued (auto-DC on transfer). This is a VistA event driver behavior that the modern UI must be aware of. Supervisor UM: 'mail message sent to PSJ-ORDERS-REINSTATED mail group when medication orders are automatically reinstated due to deletion of a patient movement.'
Rule 11: The pick list is the unit dose dispensing workflow. Pre-calculated doses per patient per administration time, printed for pharmacy technicians to fill. This is a pharmacy operations workflow, not a clinical workflow.
Rule 12: IV labels must include barcodes for BCMA scanning. Zebra bar code printer support is documented in PSJ TM. Med ROUTE prints on IV labels for bar-code enabled printers.
Rule 13: The IV room concept is important — a facility may have multiple IV rooms, each with its own label printer and report devices. The IV pharmacist selects their IV room at session start. The modern UI should support IV room selection.
Rule 14: Pharmacist Interventions and CPRS Provider Overrides must be visible in the order profile. PSJ TM documents entry points GETPROVL^PSGSICH1 (provider overrides) and INTRDIC^PSGSICH1 (pharmacist interventions). These provide clinical decision-support context.
