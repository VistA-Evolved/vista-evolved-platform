# 1. Foundation: The Configuration Backbone of All Pharmacy
National Drug File (NDF) and Pharmacy Data Management (PDM) are the two packages that CONFIGURE how pharmacy works. Every other pharmacy volume (5, 5B, 5C, 5D) depends on the data maintained by these packages. If NDF/PDM data is wrong, then:
(a) CPRS medication ordering shows wrong drugs, wrong dosages, or wrong orderable items
(b) Drug-drug interaction checking fails to detect dangerous combinations or creates false positives
(c) Pharmacy dispensing uses incorrect drug names, incorrect packaging, incorrect pricing
(d) BCMA barcode scanning cannot match medications because the drug file entries are misconfigured
(e) Controlled substance tracking has incorrect DEA schedule designations
(f) Billing/ECME pharmacy claims use wrong NDC codes, resulting in claim rejections
This volume is therefore the PREREQUISITE for all pharmacy operational volumes. It should be implemented BEFORE or concurrent with outpatient pharmacy, inpatient pharmacy, BCMA, and controlled substances.
## 1.1 Two-Level Drug Data Architecture
VistA pharmacy uses a TWO-LEVEL drug data architecture: national data maintained centrally by VA Pharmacy Benefits Management (PBM) and distributed via monthly NDF updates, and local data maintained by each facility's pharmacy staff to match national products to their local formulary.

The data structures are sound — only the national data source changes.

## 3.2 RXCFG-03: Orderable Item Management
### Purpose
Manage PHARMACY ORDERABLE ITEM #50.7 — the file that determines WHAT CPRS SHOWS when a clinician orders a medication. This is the bridge between pharmacy's drug catalog and the clinician's ordering experience. When a doctor types 'Lisinopril' in CPRS, the orderable item determines which dosage forms, strengths, and routes appear in the order dialog.
Source: PDM TM: 'Orderable Item maintenance. Master File Update sent to CPRS when Orderable Item fields are edited.' This means changes to orderable items take effect in CPRS via the Master File Update protocol, not immediately. After editing an orderable item, the pharmacy must wait for the Master File Update to propagate before clinicians see the change.

### Master File Update to CPRS
When an orderable item is edited, PDM sends a Master File Update message to CPRS via HL7 protocol. This is how pharmacy configuration changes propagate to the ordering system. PDM TM: 'sends the Master File Update to CPRS on that Orderable Item. This type of update occurs when the fields listed below are edited.' The update includes: formulary status, OI name, OI type, linked dispense drugs. CPRS caches orderable item data — changes are not visible to clinicians until the Master File Update processes.
CPRS DEPENDENCY: If orderable items are misconfigured, clinicians cannot order medications correctly. If the Master File Update fails to propagate, CPRS shows stale data. The modern UI must provide visibility into Master File Update status for pharmacy administrators.

DRUG FILE EDITING IS THE HIGHEST-RISK CONFIGURATION ACTION IN PHARMACY. Incorrect drug file data causes cascading failures across all pharmacy packages, CPRS ordering, BCMA scanning, billing claims, and controlled substance tracking. Every drug file edit should be reviewed before saving. Consider a review/approval workflow for high-risk field changes (DEA scheduling, formulary status, possible dosages).

# 5. Rules for Design and Engineering
Rule 1: DRUG #50 is the most heavily referenced file in all pharmacy. Changes to #50 cascade to: CPRS ordering, outpatient dispensing, inpatient dispensing, BCMA scanning, controlled substance tracking, billing claims, inventory management. EXTREME CAUTION on all writes.
Rule 2: NDF matching requires segregation of duties: one person matches, a DIFFERENT person verifies. NDF UM: 'To ensure accuracy one individual should match the drugs and another should verify.' The modern UI must enforce this.
Rule 3: Orderable Item changes trigger Master File Updates to CPRS via HL7 protocol. Changes are NOT immediately visible to clinicians — they propagate via the update. The modern UI should show Master File Update status.
Rule 4: Possible dosages MUST be configured for dosage order checks to work. PDM UM: 'data needs to be populated if Dosage checks are to be performed.' Without dosage data, CPRS cannot warn clinicians about incorrect doses.
Rule 5: High-risk medications have supra-therapeutic dosage protection. PSS*1*155 prevents auto-creation of dangerously high dosages using VA PRODUCT #50.68 fields. The modern UI must respect these protections.
Rule 6: CS Federal Schedule in VA PRODUCT #50.68 must match DEA Special Handling in DRUG #50. Mismatch generates warning. PDM UM: 'CS Federal Schedule warning if does not match DEA Special Handling.' Mismatches cause controlled substance tracking errors.
Rule 7: Monthly NDF updates are CRITICAL and must be processed promptly. New drugs, formulary changes, safety updates, and schedule changes arrive via monthly PSN*4* patches. Delayed processing means the site operates with outdated national drug data.
Rule 8: CMOP marking changes the dispense unit to VA standard and may overwrite the local drug name with VA Print Name. PDM UM: 'If you mark to transmit to CMOP, it will replace your Dispense Unit with the VA Dispense Unit.' This is irreversible for the display name.
Rule 9: Sensitive drug flag 'U' (PSS*1*131) identifies drugs for HIV, substance abuse, alcohol abuse, and sickle cell. These drugs require signed patient consent before information can be disclosed. The modern UI must enforce disclosure restrictions for 'U'-flagged drugs.
Rule 10: INTERNATIONAL ADAPTATION: The NDF architecture assumes VA PBM distributes national drug data. For international deployment, an equivalent national drug database must be substitutable: UK dm+d (Dictionary of Medicines and Devices), Australian AMT (Australian Medicines Terminology), WHO ATC (Anatomical Therapeutic Chemical) classification. The local DRUG #50 structure and the match/verify/merge workflow are universal — only the national source changes.
Rule 11: The Drug Enter/Edit screen must show the NDF match status for each drug: Matched (green), Unmatched (amber), Match Pending Verification (yellow). Unmatched drugs may have incomplete classification, missing ingredients (breaking allergy checking), and incorrect national formulary status.
Rule 12: Administration schedules (#51.1) must have FREQUENCY IN MINUTES populated for dosage checking to calculate daily dose. If frequency is missing, the system cannot determine whether a total daily dose exceeds safe limits.
Rule 13: IV Additive and IV Solution files (#52.6, #52.7) must have correct Orderable Item pointers to #50.7 for IV ordering in CPRS and Inpatient Medications (Volume 5B) to work correctly. PDM TM documents the cross-reference update logic that maintains these pointers.
Rule 14: Warning labels are a patient safety feature. Label assignment is per drug in DRUG #50. Labels print in defined order on prescription labels and CMOP output. Missing warning labels (e.g. 'Do not drink alcohol' for metronidazole) are a patient safety gap.
