# Section A: Blood Bank (VBECS)
## A.1 Architecture: VBECS Is a SEPARATE Application
CRITICAL architectural point: Blood Bank in VistA is NOT a traditional VistA M package like Pharmacy or Lab Chemistry. VBECS (VistA Blood Establishment Computer Software) is a SEPARATE Windows-based application hosted on Microsoft Azure for Government (MAG). It has its own SQL database, its own GUI, and its own FDA medical device regulatory status. It connects to VistA via VistALink (for patient/order data) and HL7 (for message exchange).
VBECS V2.4.0 User Guide (July 2025): 'VBECS is a Windows-based application hosted on a server. Users access VBECS through a graphical user interface and other VA systems interface with VBECS through Health Level 7 (HL7) and VistALink.' 'VBECS operates in accordance with FDA regulations regarding change to a medical device.'
This means VistA Evolved's modern web client does NOT directly replace VBECS — it INTEGRATES with VBECS. The modern UI provides the clinician-facing ordering and result viewing; VBECS provides the blood bank laboratory processing.

## 3. ABO/Rh Typing
Blood Bank Tech
Determine patient's ABO group (A, B, AB, O) and Rh (D) type. Forward and reverse typing. Compare to historical ABO/Rh. VBECS: 'historical ABO/Rh display includes date and division.' Discrepancy triggers alert and investigation.
VBECS Patient Testing
## 4. Antibody Screen
Blood Bank Tech
Screen patient plasma for unexpected red cell antibodies using reagent red cells. If positive: identify specific antibody (anti-Kell, anti-Duffy, anti-Kidd, etc.). Antibody ID determines which donor units are compatible.
VBECS Patient Testing
## 5. Crossmatch
Blood Bank Tech
Match specific donor unit to patient. Electronic crossmatch (if no antibodies and valid current ABO/Rh) or Serologic crossmatch (physical testing if antibodies present). VBECS: 'Rules for Electronic and Serologic Crossmatch.' VBECS compares interpretations with truth tables.
VBECS Blood Units: Select Units
## 6. Issue
Blood Bank Tech
Issue compatible unit to requesting location (OR, ER, ICU, ward). Print Blood Transfusion Request Form (BTRF) and Caution Tag. Record issue time, destination, receiving person.
VBECS Patients: Issue Blood Components
## 7. Transfusion
Nurse at bedside
Administer blood product to patient. CRITICAL bedside verification: two-person check of patient ID (wristband), unit label, ABO/Rh compatibility. Monitor for transfusion reactions (febrile, allergic, hemolytic).
Documented in VistA clinical chart (BCMA or nursing documentation)
## 8. Reaction Workup
Blood Bank Tech
If transfusion reaction suspected: stop transfusion, return unit and post-transfusion specimen to blood bank. VBECS: 'Record a Transfusion Reaction Workup.' DAT (Direct Antiglobulin Test), visual hemolysis check, clerical check.
VBECS Patient Testing: Transfusion Reaction Workup
PATIENT SAFETY: ABO-incompatible transfusion can be FATAL within minutes. Blood Bank has the MOST RIGOROUS identity verification, testing, and documentation requirements of any laboratory section. Every step requires positive patient identification, specimen/unit labeling verification, and double-checking. VBECS enforces many of these checks computationally (truth table comparison, electronic crossmatch eligibility rules, ABO/Rh discrepancy alerts).

# 4. Rules for Design and Engineering
Rule 1: VBECS is an FDA-REGULATED MEDICAL DEVICE. VistA Evolved does NOT replace VBECS — it integrates with it. The modern UI handles clinical ordering (BB-01) and result viewing (BB-02, BB-03, BB-04), while VBECS handles all laboratory testing, crossmatching, and inventory management. Any modification to how data flows to/from VBECS requires FDA change control review.
Rule 2: ABO-INCOMPATIBLE TRANSFUSION IS A 'NEVER EVENT.' Every blood bank integration point must enforce positive patient identification. The order must match the patient, the specimen must match the patient, the crossmatch must match the patient, and the bedside transfusion must verify the match. The modern UI must display prominent ABO/Rh information whenever blood products are being ordered or administered.
Rule 3: ISBT 128 is the international standard for blood product labeling and identification. Blood product codes, donation IDs, and component descriptions must use ISBT 128 encoding. The modern UI should display ISBT 128 information alongside human-readable text.
Rule 4: The MAXIMUM SURGICAL BLOOD ORDER SCHEDULE (MSBOS) defines how many blood units should be crossmatched for each type of surgical procedure. This prevents over-ordering (waste) and under-ordering (emergency shortages). The modern UI should reference the facility's MSBOS when blood is ordered for scheduled surgery (Vol 12 integration).
Rule 5: Anatomic Pathology reports are LEGAL MEDICAL DOCUMENTS that drive cancer treatment decisions. A surgical pathology report stating 'positive margins' means the surgeon may need to take the patient back to the OR. An incorrect diagnosis can lead to wrong treatment (unnecessary chemotherapy or missed cancer). Report verification/sign-out (AP-05) is an IRREVERSIBLE action per the Audit/Irreversibility Matrix (C/L3). Amendments create addenda, not edits.
Rule 6: CAP SYNOPTIC CANCER REPORTING is an accreditation requirement. Cancer resection specimens MUST be reported using structured CAP protocol templates with all required data elements. The modern UI must support: (a) template selection per tumor type, (b) structured data entry for each required field, (c) synoptic formatting in the final report, (d) CAP template version tracking (CAP updates protocols quarterly).
Rule 7: SNOMED coding in AP is used for CASE RETRIEVAL, not billing. Pathologists assign SNOMED codes (Topography + Morphology + Function + Disease + Procedure) to each case for searchability. The SNOMED search capability is a powerful research and quality tool. The modern UI should provide efficient SNOMED code entry with typeahead search.
Rule 8: FROZEN SECTION is a TIME-CRITICAL intraoperative consultation. The surgeon sends a specimen from the OR to pathology, and the pathologist must provide a preliminary diagnosis within 15-20 minutes while the patient is still on the operating table. The modern UI must support rapid frozen section communication: alert pathologist of incoming frozen, display result to surgeon, track frozen-to-permanent correlation (quality metric).
Rule 9: Digital Pathology (whole slide imaging) is an emerging technology that VistA Imaging supports (DICOM SM — Slide Microscopy modality). The AP workstation (AP-03) should support digital slide viewing alongside microscope-based examination. Digital slides are VERY LARGE (1-10+ GB per slide) and require specialized image viewers.
Rule 10: Blood product inventory management includes: receiving from blood supplier (Red Cross, community blood center), ABO/Rh confirmation on receipt, storage temperature monitoring, expiration tracking, unit modifications (irradiation, leukoreduction, thawing FFP), transfer between facilities. The inventory dashboard (BB-06) should provide real-time visibility with expiration alerts.
Rule 11: INTERNATIONAL ADAPTATION: Blood banking standards are internationally consistent (AABB standards recognized worldwide, ISBT 128 is international). Anatomic pathology practices vary less by country than billing or pharmacy — the same surgical pathology workflow applies globally. CAP synoptic protocols are used internationally. What may vary: local accreditation requirements (CAP US, UKAS UK, NATA AU), local tumor registry reporting requirements (NAACCR US, Cancer Research UK, AIHW AU).
Rule 12: Tenant Profile Matrix visibility: Blood Bank = PRIMARY for Profile A (Full Hospital), HIDDEN for all other profiles. Anatomic Pathology = PRIMARY for Profile A, OPTIONAL for Profile F (Standalone Lab if they do AP), HIDDEN for others. These are hospital-level sub-specialties not relevant to clinics, pharmacies, or standalone imaging.
